Local Search

In local search, we are unconcerned with the path to the goal state. Here, the goal state is itself the solution.

To find it, we keep a single 'current' state and try to improve it.

Hill-climb

"Like climbing Everest in thick fog with amnesia."

Hill-climb finds a local maximum; it iterates its state by simply taking the route which increases the evaluation function.

This is its weakness, though - it can get stuck in local maxima, which sucks if we want to find a global maximum.

n-queens

Remember the n-queens problem? You have to place nnn﻿ queens on an n×nn\times nn×n﻿ board such that none of them can take each other.

You can try hill-climb on this. We evaluate the boards state via the number of queens attacking each other - the fewer the better.

However, for a random initialisation, hill-climb gets stuck on 8-queens 86% of the time. That kinda sucks, but could be passable depending on what you were looking for from your solution.

Interestingly, increasing nnn﻿ does not have a large effect on the percentage of times that hill-climb gets stuck on the nnn﻿-queens problem.

The lecturer framed this as the state space increasing exponentially, but also the number of answers would increase exponentially.

Simulated Annealing

Simulated annealing is an extension of hill-climb which seeks to stop it from getting stuck so much.

Idea: escape local maxima by allowing bad moves, but gradually decrease the likelihood of making those bad moves as time goes on.

It can be proven that, if the "temperature" decreases slowly enough, simulated annealing will find a global optimum with probability approaching 1.

This is ridiculously slow, however. With slow enough cooling, you end up just visiting every possible solution (which is where the approaching 1 comes from), so you might as well just use exhaustive search.

For situations where you need a good estimate, simulated annealing is widely used! VLSI layout, airline scheduling, neural network training, etc.

Genetic algorithms

Local beam search

This is a fairly obscure and unused algorithm, but it leads well into genetic algorithms.

We keep track of kkk﻿ states rather than just one - exploit parallelism.

  • Start with kkk﻿ randomly generated states.
  • At each iteration, all the successors of all kkk﻿ states are generated.
  • If any one is a goal state, we stop. Otherwise, we select the kkk﻿ best successors from the complete list and repeat.

This seems slightly naive, but there are some clever ways to optimise it. The gist of it is that we put more computational effort into the better-looking solutions.

Genetic algorithms

The idea is that a successor state is generated by combining two parent states. A state is represented as a string over a finite alphabet.

We start with kkk﻿ random states and produce the next generation of states through selection, crossover and mutation.

Turns out genetic algorithms aren't actually much better than simulated annealing in general, according to the lecturer. There are some uses for them though apparently.
