Best-first search

Idea: we use an evaluation function f(n)f(n)f(n)﻿ for each node.

This gives us an estimate of "desirability"; we choose to expand the most desirable unexpanded node.

Implementation: we order the nodes in the fringe (unexpanded) in decreasing order of desirability.

Greedy best-first search

In greedy best-first search, the evaluation function h(n)h(n)h(n)﻿ is a heuristic estimate of cost from nnn﻿ to the goal. This is how we pick which node to expand.

For example, in the Romania Map problem which you've probably forgotten by now, you could pick hSLD(n)h_{SLD}(n)hSLD​(n)﻿ to be the straight-line distance (as the crow flies) from nnn﻿ to Bucharest.

Properties of greedy best-first search

Complete? No, it can get stuck in loops (like DFS) if the heuristic is bad.

Time? O(bm)O(b^m)O(bm)﻿ - a good heuristic can give dramatic improvement, though.

Space? O(bm)O(b^m)O(bm)﻿ - all the nodes are kept in memory.

Optimal? Nope.

A* search

Idea: avoid expanding paths which are already expensive.

Our evaluation function is f(n)=g(n)+h(n)f(n)=g(n)+h(n)f(n)=g(n)+h(n)﻿ where

g(n)g(n)g(n)﻿ is the cost so far to reach nnn﻿

h(n)h(n)h(n)﻿ is the estimated cost from nnn﻿ to the goal using a heuristic

This gives us a function f(n)f(n)f(n)﻿ which is the estimated total cost of the path through nnn﻿ to the goal.

Admissible heuristics

A heuristic h(n)h(n)h(n)﻿ is admissible if, for every node nnn﻿:

h(n)≤h∗(n)h(n)\leq h^*(n)h(n)≤h∗(n)﻿

where h∗(n)h^*(n)h∗(n)﻿ is the true cost to reach the goal state from nnn﻿.

That is, it's optimistic. It never overestimates the cost to reach the goal.

Optimality of A*

If the heuristic you're using for A* is admissible, then A* on tree-search is optimal.

However, this doesn't hold for graph-search straight away.

A* might discard the optimal path to a repeated state if it's not the first one generated.

If the cost for any action was constant, this wouldn't be a problem. Alternatively, it wouldn't be a problem if we could guarantee that the first path found to a repeated node was the best path to that node.

We can guarantee this if we impose an extra requirement on the heuristic...

≥g(n)+h(n)=f(n)\ge g(n)+h(n)=f(n)≥g(n)+h(n)=f(n)﻿

That is, f(n)f(n)f(n)﻿ is non-decreasing along any path.

This means that the sequence of nodes expanded by A* is in non-decreasing order of f(n)f(n)f(n)﻿.

Hence, the first goal to be selected for expansion must be an optimal solution!

(since all latter nodes are at least as expensive)

Relaxed problems

It's often impractical to find an optimal solution. There are 'more accurate' heuristics which are not strictly admissable.

So, how do you create a heuristic?

You can conjure a relaxed problem. This is a problem with fewer restrictions on the actions.

The cost of an optimal solution to a relaxed problem is an admissible heuristic for the original problem.
