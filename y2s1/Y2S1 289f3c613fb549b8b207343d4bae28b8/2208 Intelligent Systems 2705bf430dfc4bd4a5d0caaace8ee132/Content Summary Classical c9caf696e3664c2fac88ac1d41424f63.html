Blind search

Blind search is typically performed using tree search algorithms.

The main idea is to generate successors of already-explored nodes, called expanding the node.

Blind search is performed offline - that is, all information has to be given at the start.

Qualities of blind search strategies

  • Completeness: does it always find a solution if there is one?
  • Optimality: does it always find a solution which has the least “cost”?
  • Time complexity: total number of nodes generated
  • Space complexity: highest number of nodes kept in memory at any point in the algorithm

Factors of complexity: b, d and m

  • bbb﻿ is the maximum branching factor of the tree - how many nodes are generated when you expand a node?
  • ddd﻿ is the depth of the least-cost solution.
  • mmm﻿ is the maximum depth of the search space - this can be infinite.

Blind search strategies

BFS

BFS expands the shallowest unexpanded node. This can be seen as looking at the “fringe” of the tree - the lowest unexpanded nodes - as a queue. New successors go at the end of the queue, so the shallower nodes are expanded first.

BFS is:

  • Complete as long is the branching factor is finite (and there are no loops).
  • Optimal as long as the cost per step is constant.

Complexities:

  • Time: O(bd+1)O(b^{d+1})O(bd+1)﻿ - at each step, bbb﻿ nodes are generated, up until the shallowest solution is found at ddd﻿.
  • Space: O(bd+1)O(b^{d+1})O(bd+1)﻿ - every node stays in memory.

As you can see, BFS suffers from a space problem.

DFS

DFS expands the deepest unexpanded node. When considering the “fringe” of the tree, look at it like a stack. New successors go at the front, so it’ll go down one branch at a time.

DFS is:

  • Complete as long as the maximum depth is finite (and there are no loops).
  • Not optimal.

Complexities:

  • Time: O(bm)O(b^m)O(bm)﻿ - at each step, bbb﻿ nodes are generated, up until the maximum depth of the search space mmm﻿. This is often awful.
  • Space: O(bm)O(bm)O(bm)﻿ - at each step, bbb﻿ nodes are generated. Since it’s a stack, we only have mmm﻿ of these. That is, we only need to remember the current branch. This is linear!

Time complexity is terrible if the shallowest solution is much shallower than the total depth.

Iterative-deepening DFS

Iterative-deepening is a depth-limited DFS which iteratively increases its maximum depth until the solution is found.

Iterative-deepening search is:

  • Complete.
  • Optimal as long as the cost per step is constant.

Complexities:

  • Time: O(bd)O(b^d)O(bd)﻿ - similar reasoning to normal DFS, except we typically stop at the solution rather than the bottom of the tree.
  • Space: O(bd)O(bd)O(bd)﻿ - as above.

Iterative deepening is almost always better than regular DFS.

DFS is better in specific situations, like where all the solutions are at the bottom of the tree - see constraint satisfaction problems.

Heuristic blind search

Best first search means that we use an evaluation function for each node, which gives us an estimate of desirability.

This means we can order the nodes in the fringe by desirability.

Greedy best-first

Essentially, order the nodes in the fringe by some heuristic evaluation function h(n)h(n)h(n)﻿.

For example, you could use Euclidean distance or Manhattan distance. (Euclidean is straight-line distance, while Manhattan is the shortest sum of all actual paths between two points).

  • Not complete - can get stuck in loops with a bad heuristic
  • Not optimal - the solution depends more on the heuristic than the actual costs

Time and space of O(bm)O(b^m)O(bm)﻿. A good heuristic can improve time dramatically, though.

A*

The idea of A* is to avoid expanding paths which are already expensive.

Hence, we use the evaluation function is f(n)=g(n)+h(n)f(n)=g(n)+h(n)f(n)=g(n)+h(n)﻿ where:

g(n)g(n)g(n)﻿ is the cost so far

h(n)h(n)h(n)﻿ is the heuristic cost from nnn﻿ to the goal

A heuristic is admissible if it is always less than or equal to the true cost to the goal. That is, it’s optimistic.

A heuristic is consistent (or monotonic) if its estimate is always less than or equal to the cost of reaching a neighbour plus the estimate from that neighbour. That is, as the name suggests, the heuristic is always increasing.

  • Complete unless there are infinitely many nodes which have a cost which is estimated by the heuristic to be less cost than the goal.
  • Optimal on tree-search if the heuristic is admissible. Optimal on graph search if the heuristic is admissible AND consistent.

One method to create heuristics is to consider a relaxed problem - the cost of an optimal solution to a relaxed problem is an admissible heuristic for the original problem.

Local search

In local search problems, we don’t care about the path to the goal state - we only care about the goal state itself.

Hill-climb and simulated annealing

Hill-climb simply takes the route which increases its evaluation function.

It can get stuck in local maxima, though.

Simulated annealing is an extension of hill-climb where you make bad moves on purpose, but gradually decrease the probability of making those bad moves over time.

Simulated annealing, when the temperature decreases slowly enough, will find a global optimum with probability approaching 1. You might as well just use exhaustive search at that point, though.

Simulated annealing is actually used a fair bit irl.

Genetic algorithms

The gist of it is that we start with kkk﻿ randomly generated states, generate the successors to those states, check if any of them are the solution - if not, select the kkk﻿ best successors and repeat.

The above is called local beam search and is a naive approach to genetic algorithms.

We can use selection, crossover and mutation to improve this.

A state in a genetic algorithm is represented as a string over a finite alphabet.

Turns out this isn’t much better than simulated annealing lmao.

Constraint satisfaction

The main difference here is that state isn’t a black box - it’s defined by a set of variables, each with values from a domain.

The goal test is a set of constraints.

Graph-colouring and nnn﻿-queens are examples of CSPs.

CSPs are a simple example of a formal representation language. It allows useful general-purpose algorithms with more power than standard search algorithms.

For a basic approach, we have:

Initial state as the empty assignment of values.

Successor function as “assigning a value to an unassigned variable which doesn’t conflict with the existing assignment”

Goal test as a simple check whether all variables are assigned.

If we assign only one variable at a time, this is effectively backtracking search.

Improving backtracking

Most constrained variable: choose the one with the fewest legal values - this is most likely to prune the search tree.

Most constraining variable: a useful tiebreaker. Choose the one with the most constraints on remaning values - like the node connected to the most other uncoloured nodes in graph-colouring.

Least constraining value: Choose the value which rules out the fewest other values in neighbouring variables.

Forward checking: keep track of remaining legal values for unassigned variables. Terminate the search when any variable has no legal values.

Arc consistency: an arc between two variables XXX﻿ and YYY﻿ is consistent if, for each value of XXX﻿, there is some value of YYY﻿ that we can assign such that it doesn’t violate the constraints.

If an arc is inconsistent, we can remove all values of XXX﻿ for which there’s no corresponding value in the domain of YYY﻿ in order to make the arc consistent.

This helps constrain the search space.

Local search in CSP

Local search needs a complete state to work.

To apply local search to CSPs, we allow states with constraint violations - operators reassign variable values.

This lets us use min-conflicts heuristics to apply hill-climb etc.

This is often effective in practice. Works well on nnn﻿-queens, at least!

Minimax

Minimax is focused on adversarial game playing, which is a traditional focus of classical AI.

MAX is usually the player, and picks the moves whose heuristic value is the best.

MIN is usually the opponent, and picks the moves which minimise MAX’s advantage.

We go through a tree of moves - each layer of the tree is assigned to a player, since the turns alternate.

We search to a given ply - it’s a depth-limited search - and evaluate the heuristic for the leaf nodes.

The internal nodes propagate the values of the heuristic back upwards, which lets you know what route to pick at the top.

  • Complete for a finite game tree
  • Optimal for an optimal opponent
  • Time of O(bm)O(b^m)O(bm)﻿ (this is just DFS)
  • Space of O(bm)O(bm)O(bm)﻿ (DFS again)

Quiescent search

A bad evaluation function can lead to wild swings in behaviour - see evaluating chess positions on material only.

We look for stable - or quiescent - positions. We can also expand nodes in nonquiescent positions until a stable solution is reached.

Alpha-beta pruning

The main idea is that we keep track of the worst and best any given branch can do (α\alphaα﻿ and β\betaβ﻿ respectively).

If the best we can do on the current branch is less than or equal to the worst we can do elsewhere, what’s the point in carrying on here?

Note that alpha-beta pruning depends on the direction in which the tree is traversed.

The ordering of subtrees has a large effect on the amount of pruning.

Alpha-beta and minimax

Alpha-beta is guaranteed to give the same values as minimax, while pruning subtrees which don’t need to be searched.
