Summary

In a standard search problem, state is a black box. It could be any data structure which supports the successor function, a heuristic function and a goal test.

In a Constraint Satisfaction Problem (CSP), state is defined by variables XiX_iXi​﻿ with values from a domain DiD_iDi​﻿.

The goal test is a set of constraints specifying allowable ranges or combinations of values.

Examples include graph-colouring, N-queens, etc.

CSPs are a simple example of a formal representation language. It allows useful general-purpose algorithms with more power than standard search algorithms.

CSP varieties

Discrete variables:

Finite domains, with nnn﻿ variables. Includes boolean CSPs - like SAT.

Infinite domains, using integers and strings etc. Includes job scheduling.

Continuous variables:

For example, the start/end times for Hubble observations.

These (often?) have linear constraints solvable in polynomial time by linear programming.

There are various types of constraint - unary, binary, higher-order.

Standard search formulation

Let’s start with a straightforward approach, then fix it.

States are defined by the values assigned so far:

Initial state is the empty assignment of values {  }\{~~\}{  }﻿.

Successor function is “assign a value to an unassigned variable that doesn’t conflict with the current assignment”.

Goal test checks whether all variables are assigned (the assignment is complete).

This is the same for all CSPs.

Every solution appears at depth nnn﻿ with nnn﻿ variables, so we’ll want a depth-first approach.

b=(n−l)db=(n-l)db=(n−l)d﻿ at depth lll﻿, hence #leaves=n!dn\#leaves=n!d^n#leaves=n!dn﻿ even though there are only dnd^ndn﻿ assignments.

This sucks!

Backtracking search

Variable assignments are commutative - assigning WA red and then assigning NT green is the same as assigning NT green then WA red.

Hence, we only need to consider assignments to a single variable at each node.

b=db=db=d﻿ with dnd^ndn﻿ leaves.

Depth-first search with single-variable assignments is effectively just a backtracking search!

This is the basic uninformed algorithm for CSPs - it can solve N-queens for n≈25n\approx 25n≈25﻿.

What’s the difference?

Conceptually, basic depth-first and the backtracking approach are very similar. In DFS, however, we assumed that a node was expanded by applying all possible actions to add nodes to the fringe.

Backtracking search just applies one action to produce one successory node and continues on down - then returns later if necessary.

Improving backtracking

What variable should be assigned next? In what order should its values be tried? Can we detect inevitable failure early?

Most constrained variable

We choose the variable with the fewest legal values, since these are the variables which are most likely to prune the search tree.

Most constraining variable

A useful tiebreaker.

The most constraining variable is the variable with the most constraints on remaining values - like the node connected to the most other uncoloured nodes in graph-colouring.

The variable involved in the most constraints is most likely to cause a failure, so we want to cause a failure early to prune the search tree.

Least constraining value

The least constraining value is the one which rules out the fewest values in the neighbourning variables, to leave other variables as open as possible.

Forward checking

Idea: keep track of remaining legal values for unassigned variables. Terminate the search when any variable has no legal values.

An arc X→YX\rarr YX→Y﻿ is arc consistent if, for each value of XXX﻿ in the domain of XXX﻿, there is some value of YYY﻿ we can assign such that doesn’t violate the constraints.

If an arc is not arc consistent, all values of XXX﻿ for which there’s no corresponding value in the domain of YYY﻿ may be deleted from the domain of XXX﻿ to make the arc consistent.

This does additional work to constrain values and detect inconsistencies.

Local search for CSP

Local search tactics like hill-climb and simulated annealing typically work with “complete” states - i.e. all variables are assigned.

To apply this to CSPs, we allow states with constraint violations. Operators reassign variable values.

To select a variable, we randomly select any conflicted variable.

To select a value, we use the min-conflicts heuristic - choose the one which violates the fewest constraints. Hill climb, babey!

Iterative min-conflicts

While we’re not at a solution:

Choose a random conflicted variable.

Pick a value of this that has minimum conflicts.

This works surprisingly well on N-queens!

Summary

CSPs are a special kind of problem with states defined by values of a fixed set of variables. The goal test is defined by constraints on those assignments.

Backtracking here is equivalent to DFS with a single variable assigned per node.

Variable ordering and value selection heurisitcs help significantly.

Forward checking prevents assignments which guarantee later failure.

Constraint propagation does additional work to constrain values and detect inconsistencies.

Iterative min-conflicts is often effective in practice.
