Converting CFG and Pushdown Automata

We can go from a CFG to a single-state PDA: this shows that we can find a PDA for any CFG which recognises the same language.

We can also go from a single-state PDA to a CFG, as well as any PDA to a single-state PDA.

This means we can do:

PDA ⇝\rightsquigarrow⇝﻿ one-state PDA ⇝\leadsto⇝﻿ CFG

Clearly, PDA and CFGs have the same descriptive power.

The PDA will have only one state qqq﻿ and will accept by empty stack.

The construction

Let G=(N,Σ,P,S)G=(N,\Sigma,P,S)G=(N,Σ,P,S)﻿. The required PDA is M=({q},Σ,N,δ,q,S,∅)M=(\{q\},\Sigma,N,\delta,q,S,\varnothing)M=({q},Σ,N,δ,q,S,∅)﻿.

We make the following conversions:

  • qqq﻿ is the only state.
  • NNN﻿, the set of GGG﻿'s non-terminals, is the stack alphabet.
  • Σ\SigmaΣ﻿, the set of GGG﻿'s terminals, is the input alphabet.
  • SSS﻿, the start symbol, is the initial stack symbol.
  • No need for final states (there's only one state and we accept via empty stack)
  • δ\deltaδ﻿ is constructed as follows:
