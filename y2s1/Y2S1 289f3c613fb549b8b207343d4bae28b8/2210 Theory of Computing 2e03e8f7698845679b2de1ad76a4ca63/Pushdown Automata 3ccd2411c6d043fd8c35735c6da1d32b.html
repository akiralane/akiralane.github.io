Pushdown Automata

One way to look at NFA is as a tape machine, reading from the tape and moving accordingly.

For pushdown automata, we add access to a stack. The stack can hold an unbounded number of symbols.

The stack language is still finite though - a crucial component of models of computation is that their description is finite.

Further, there is never a point in computation where the stack holds an infinite number of symbols - this is why we call it unbounded instead of infinite.

At any time, the automaton can read from the stack and push or pop symbols according to what it sees.

The addition of the stack allows PDA to recognise some irregular languages (specifically CFLs, but we’ll come to that later). Here's an example:

L={anbn∣n≥1}L=\{a^nb^n|n \geq 1\}L={anbn∣n≥1}﻿

There's a PDA which recognises the above:

If aaa﻿ is read, then we push an aaa﻿ onto the stack.

If bbb﻿ is read, then we pop an aaa﻿ off the stack.

We accept when we finish consuming symbols AND the stack is empty of aaa﻿'s AND the final state is reached.

Otherwise, we reject.

Γ\GammaΓ﻿ is the stack alphabet,

δ\deltaδ﻿ is the transition relation,

s∈Qs\in Qs∈Q﻿ is the start state like usual,

F⊆QF\sube QF⊆Q﻿ is the set of final states.

Q,Σ,Γ,FQ,\Sigma,\Gamma,FQ,Σ,Γ,F﻿ are all finite sets.

Configurations

In order to define a notion of computation, we want a concept which describes the current "state" of the entire PDA. This is a configuration.

A configuration is an element of Q×Σ∗×Γ∗Q\times \Sigma^* \times \Gamma^*Q×Σ∗×Γ∗﻿ and has the form (q,w1…wk, v1…vm ⊥)(q,w_1\dots w_k,~v_1\dots v_m~\bot )(q,w1​…wk​, v1​…vm​ ⊥)﻿.

Transitions

The missing piece of our notion of computation now is the idea of transitions between configurations.

We write (q,w1…wk, av1…vm ⊥)→(q′,w2…wk, bv1…vm ⊥)(q,w_1\dots w_k,~av_1\dots v_m~\bot )\to(q',w_2\dots w_k,~bv_1\dots v_m~\bot )(q,w1​…wk​, av1​…vm​ ⊥)→(q′,w2​…wk​, bv1​…vm​ ⊥)﻿ whenever:

((q,w1,b),(q′,a))((q, w_1, b),(q',a))((q,w1​,b),(q′,a))﻿

That is "in state qqq﻿, read w1w_1w1​﻿, bbb﻿ is on the stack", then "move to q′q'q′﻿, (pop bbb﻿ off the stack and) put aaa﻿ on".

as long as you remember that a transition is an element of the transition relation, which is a finite subset of (Q×(Σ∪{ε})×Γ)×(Q×Γ∗)(Q\times (\Sigma \cup \{\varepsilon\})\times\Gamma)\times (Q\times\Gamma^*)(Q×(Σ∪{ε})×Γ)×(Q×Γ∗)﻿. That seems long, but it makes sense if you look at it.

Example

Make a PDA that accepts the language {anbn∣n≥1}\{a^nb^n | n \geq 1\}{anbn∣n≥1}﻿. This is the one from the top of this page!

The idea is to count the number of aaa﻿'s we read at the start by pushing them onto the stack, and then popping one off for each bbb﻿ you read. We accept when we've reached the bottom of the stack and there aren't any bbb﻿'s left (which we'll represent using the final state).

Let:

Σ={a,b}\Sigma = \{a,b\}Σ={a,b}﻿

Γ={a,⊥}\Gamma=\{a,\bot\}Γ={a,⊥}﻿

Q={0,1,2}Q=\{0,1,2\}Q={0,1,2}﻿

s=0,F={2}s=0,F=\{2\}s=0,F={2}﻿

State 0:

If I read an aaa﻿, and the stack is empty, I push a⊥a\bota⊥﻿ onto the stack.

If I read an aaa﻿, and the stack has an aaa﻿, I push aaaaaa﻿ onto the stack.

(remember that taking a transition always pops from the stack)

If I read a bbb﻿, pop an aaa﻿ from the stack and move to state 1.

State 1:

If I read a bbb﻿, pop an aaa﻿ from the stack.

If the stack is empty, transition to the accepting state.

You can clearly see here that PDA are nondeterministic - we don't have to define all behaviour (and probably shouldn't!). Whenever we encounter undefined behaviour, we transition to the error state ∅\empty∅﻿ and therefore reject the string.
