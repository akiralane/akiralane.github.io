i.e. decide whether the language of a given DFA is empty or not

A DFA accepts strings iff reaching an accept state by traversing the arrows of the DFA is possible. To test this condition, we can design a TM which uses a marking algorithm.

Mark the start state of AAA﻿,

Repeat the following until no new states get marked:

Mark any state which has a transition coming into it from any state that's already marked.

If no accept state is marked, accept. Otherwise, reject. That is, if there's any path, we reject because we want the empty language.

Turing machines are exactly as expressive as deterministic algorithms, so we can just write out words and ignore the implementation.

i.e. decide whether a CFG accepts a certain string

We can't go through all the derivations to determine whether any is a derivation of www﻿, since there could be infinitely many. If GGG﻿ doesn't generate www﻿, that algorithm would never halt and we wouldn't be able to prove the recursiveness.

We need to ensure that the algorithm only tries finitely many derivations.

In any GGG﻿ in Chomsky normal form, any derivation of www﻿ has 2n−12n-12n−1﻿ steps, where nn n﻿ is the length of www﻿. This means that only finitely many such derivations exist!

So we can just convert GGG﻿ into an equivalent grammar in CNF and enumerate the derivations with 2n−12n-12n−1﻿ steps. Easy!

i.e. decide whether the language of a CFG is empty

We need to test whether the start variable can generate a string of terminals - a sentence, that is.

First, it should mark all the terminal symbols in the grammar.

Then, it should scan all the rules of the grammar. If it finds a rule that permits some variable to be replaced with some string of symbols, all of which are already marks, the algorithm knows that this variable can be marked too.

The algorithm continues like this until it can't mark any new variables.

If the start variable isn't marked, accept, otherwise reject.
