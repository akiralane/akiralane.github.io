Single vs Multi-tape Turing machines

It’s more convenient - and efficient - to use multitape TMs.

They’re equivalent to single-tape TMs when it comes to decidability...

...but not equivalent when it comes to complexity.

Then again, the gain is only quadratic:

Every t(n)t(n)t(n)﻿ time multi-tape Turing machine has an equivalent O(t2(n))O(t^2(n))O(t2(n))﻿ time single-tape Turing machine.

This tells us that (unlike TIME) having polynomial complexity is a fairly robust property of languages - it doesn’t depend on the choice of (deterministic!) computational model.

So, let’s go non-deterministic!

Intuition

A non-deterministic TM can be thought of as making all possible transitions from a given configuration in parallel.

The computation of a non-deterministic TM is a tree, where:

nodes are configurations (where the root is obviously the start configuration)

leaves are accept/reject configurations

branches are individual computation processes.

This is a non-physical model of computation.

Computation

A non-deterministic Turing machine:

accepts if some branch of the computation leads to an accept configuration

rejects if all branches of the computation lead to reject configurations.

That is, the machine accepts an input if there exists a sequence of “choices” which ends in an accept configuration.

A non-deterministic Turing machine is a decider if ALL branches halt on all inputs.

A decider will always either accept or reject an input (equivalent to totality of deterministic TMs).

Expressiveness

For any non-determinisitic TM, there exists a deterministic TM accepting the same language.

Idea: you can simulate it with a (multi-tape) deterministic TM which tries all branches of the non-deterministic computation breadth-first.

One tape stores the input,

one tape is used to simulate a single branch up to a certain depth

one tape is used to remember which branch is being simulated.

If this machine ever finds the accept state on one of the branches, it will accept. Otherwise, it’ll loop. See Sipser for details, apparently.

We can modify this so that the machine remembers which branches we’ve visited (on a fourth tape, for example) and reject if there are no more branches left to explore.

Hence, since D-TMs and ND-TMs have the same expressiveness:

A language is recursively enumerable iff some non-deterministic TM accepts it.

That is, they are equivalent in terms of expressiveness.

Time complexity of deciders

The time complexity of a decider is:

the maximum number of steps the decider uses on any branch of its computation on any input of length nnn﻿.

Note that the above doesn’t correspond to any typical real-world computing device, but we can use it later to defined an important class of problems!

Cost of determinism

Recall that moving from multi-tape to single-tape TM has a quadratic cost. How about moving from a ND-TM to a D-TM?

Turns out:

every t(n)t(n)t(n)﻿ time ND-TM has an equivalent 2O(t(n))2^{O(t(n))}2O(t(n))﻿ time D-TM.

Hence, non-deterministic TMs are potentially more efficient than deterministic ones, despite adding no expressive power.
