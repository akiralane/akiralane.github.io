NP-hardness and the Cook-Levin Theorem

If we can reduce all NP problems to a certain problem in polynomial time, then that problem must be ‚Äúhard‚Äù (in a kind of material sense, I think), since a polynomial time algorithm for it leads to polynomial time algorithms for all the NP problems.

The Cook-Levin Theorem says that every problem in NP can be polynomial-time reduced to SAT - not just 3COL!

SAT is at least as hard as any other NP problem.

This means that if we can find a polynomial time algorithm for SAT, not only is SAT in P, all of NP is in P and P=NP!

Summary of complexity so far (time)

Languages in P can be decided in polynomial time.

Languages in NP can be verified in polynomial time.

We don‚Äôt know whether P=NP.

Examples of NP problems include SAT, 3COL and TSP(D).

Polynomial time reduction helps us to work out which key problems to solve in order to answer P=NP.

The Cook-Levin theorem states that we can reduce any NP problem to SAT in polynomial time. All problems with this property are called NP-hard.

If a problem which is NP-hard is also in NP, then it‚Äôs called NP-complete.

If we find a polynomial time algorithm for a single NP-complete problem, we get one for all NP problems.
