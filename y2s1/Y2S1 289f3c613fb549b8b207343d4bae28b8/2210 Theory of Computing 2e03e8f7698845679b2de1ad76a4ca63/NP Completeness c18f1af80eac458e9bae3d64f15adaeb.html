Recap:

P is the class of languages for which membership can be decided quickly.

This is invariant for all deterministic models of computation!

NP is the class of languages for which membership can be verified quickly.

EXPTIME

The class EXPTIME is the set of all languages which can be decided by a deterministic Turing machine in exponential time:

EXPTIME=⋃k∈N TIME(2nk)\text{EXPTIME}=\bigcup_{k\in\N}\text{ TIME}({2^n}^k)EXPTIME=⋃k∈N​ TIME(2nk)﻿

nnn﻿ is the size of the input and kk k﻿ is a constant.

HAMPATH, SAT, 3COL are all in EXPTIME if you just use brute force to solve them.

In fact, we can prove that NP ⊆\sube⊆﻿ EXPTIME.

However, we don’t know if NP is contained in a deterministic time complexity class which is smaller than EXPTIME!

Polynomial time reduction

If we can reduce one language (or decision problem) to another, does a polynomial time algorithm for the first give us a polynomial time algorithm for the other?

It depends on how fast the reduction can be computed. If the translation algorithm runs in polynomial time, then it’s no problem! If it’s exponential, then nope.

A function is polynomial time computable if there exists a polynomial time deterministic Turing machine which halts on each input www﻿ with just f(w)f(w)f(w)﻿ on its tape.

NP-hardness and the Cook-Levin Theorem

If we can reduce all NP problems to a certain problem in polynomial time, then that problem must be “hard” (in a kind of material sense, I think), since a polynomial time algorithm for it leads to polynomial time algorithms for all the NP problems.

The Cook-Levin Theorem says that every problem in NP can be polynomial-time reduced to SAT - not just 3COL!

SAT is at least as hard as any other NP problem.

This means that if we can find a polynomial time algorithm for SAT, not only is SAT in P, all of NP is in P and P=NP!

NP-completeness for real, for real

As we’ve said above, a language BBB﻿ is NP-hard if every language AAA﻿ in NP is polynomial time reducible to BBB﻿.

If BBB﻿ is also in NP, then BBB﻿ is called NP-complete.

Summary of complexity so far (time)

Languages in P can be decided in polynomial time.

Languages in NP can be verified in polynomial time.

We don’t know whether P=NP.

Examples of NP problems include SAT, 3COL and TSP(D).

Polynomial time reduction helps us to work out which key problems to solve in order to answer P=NP.

The Cook-Levin theorem states that we can reduce any NP problem to SAT in polynomial time. All problems with this property are called NP-hard.

If a problem which is NP-hard is also in NP, then it’s called NP-complete.

If we find a polynomial time algorithm for a single NP-complete problem, we get one for all NP problems.
