Total Turing Machines

A TM is said to halt on an input if it either accepts or rejects that input.

It's possible that the machine neither accepts nor rejects - it loops.

A TM is total if it halts for all inputs.

Recursive and recursively enumerable sets

Let L(M)L(M)L(M)﻿ be the set of strings accepted by a TM MMM﻿. That is, L(M)L(M)L(M)﻿ is the language of MMM﻿. A set of strings XXX﻿ is called:

  • recursively enumerable if it is the language of some TM. X=L(M).X=L(M).X=L(M).﻿
  • co-recursively enumerable if its complement is recursively enumerable.
  • recursive is it is the language of some total TM. X=L(M)X=L(M)X=L(M)﻿ for total MMM﻿.

Recursive sets are closed under complement.

Suppose AAA﻿ is recursive - then there exists a total TM which accepts it. We can just swap the accept and reject states of the TM, just like we did for DFA. This swapped TM now accepts the complement of AAA﻿.

This works because "rejecting" and "not accepting" mean exactly the same thing for a total machine. The same cannot be said for a non-total machine, which can loop forever.

As you can intuit from the relation to TMs, every recursive set is recursively enumerable, but not vice versa.

Decidability and semidecidability

Let PPP﻿ be some property of strings. That is, for any string xxx﻿, either P(x)P(x)P(x)﻿ or ¬P(x)\neg P(x)¬P(x)﻿.

This can be something super simple, like "has a length of 2".

That is, there exists a total TM which describes all strings with the property.

The property is semidecidable iff {x∈Σ∗∣P(x)}\{x\in\Sigma^*|P(x)\}{x∈Σ∗∣P(x)}﻿ is recursively enumerable.

That is, there exists any TM which describes all strings with the property.
