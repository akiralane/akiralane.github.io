Regular languages / CFL and the class P

If we’re only interested in a coarse classification of languages by complexity:

we can view polynomial running time as “feasible” or “tractable”

we can view exponential running time as “infeasible”.

Informally

The class P is the set of all decision problems which can be solved in polynomial time by a deterministic machine.

It doesn’t matter what kind of machine, as long as the model is Turing-complete. P is the same of all of them.

Formally

P=⋃k∈N  TIME(nk)\text{P}=\bigcup_{k\in\N}~~\text{TIME}(n^k)P=⋃k∈N​  TIME(nk)﻿

That is, P is the class of languages that can be decided by a deterministic, single tape Turing machine in polynomial time.

Importance

Robustness: it’s invariant for all deterministic models of computation. This includes models which capture running time on actual computers!

This means that it provides a mathematical model for problems which are realistically solvable on a computer!

Improving machine speed will not change P.

Examples

PATH is the decision problem of finding the existence of a directed path between two nodes in a directed graph.

The above is a polynomial time algorithm for PATH. Hence, PATH ∈\in ∈﻿ P.

HAMPATH is the decision problem of whether or not such a path exists which goes through each node exactly once (that is, a Hamiltonian path).

We don’t know if there exists a polynomial time algorithm for HAMPATH. We can, however, given a candidate for a Hamiltonian path, verify whether it’s valid in polynomial time.

This’ll be important later.

Regular languages / CFL and the class P

Every regular language is in P. We can simply simulate the running of a DFA, which is fairly straightforward. Since we’re just traversing strings, it can only every be polynomial.
