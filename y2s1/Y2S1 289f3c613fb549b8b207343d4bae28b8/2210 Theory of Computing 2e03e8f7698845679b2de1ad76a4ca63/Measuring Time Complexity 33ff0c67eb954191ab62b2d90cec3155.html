Complexity class TIME

Computability was concerned with which problems are solvable with a computer and which aren’t.

Complexity theory, however, deals with whether those problems that are solvable are actually reasonably solvable in practice by classifying their usage of time and/or space.

Precise terminology

A problem is a problem. It has a set of possible inputs and prescribed corresponding outputs.

An algorithm is a procedure for solving a problem.

A problem instance is a possible input to a given problem.

e.g. sort the list “21 3 63 2 28 30”

Decision problems

A computational problem is a decision problem if the output for any given instance is either “yes” or “no”.

We can think of the set of all “yes” instances as forming a language.

Is x prime? → The language of all prime numbers.

Does this graph G admit a Hamiltonian path? → The language of all graphs which admit Hamiltonian paths.

Is the following text a C program? → The language of all C programs (also known as C).
