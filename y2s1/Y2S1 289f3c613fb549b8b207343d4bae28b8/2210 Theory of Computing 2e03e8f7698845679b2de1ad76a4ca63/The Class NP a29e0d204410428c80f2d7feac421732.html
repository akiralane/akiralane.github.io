Some notes on encoding

Previously, we’ve proved that both PATH and RELPRIME are in P.

However, for PATH we gave an algorithm which was polynomial in the number of nodes in the graph, while for RELPRIME we gave an algorithm which was polynomial in the length of the input!

Why the difference?

For PATH, any reasonable encoding of the graph is polynomial in the number of nodes.

So, if we prove that our algorithm runs in polynomial time in the number of nodes, it will also run in polynomial time in the length of the encoding of the nodes.

Clearly, this applies to other encodings as well.

Polynomial time verifiers

Remember what we said about verifying HAMPATH last time? We’re coming back to that now.

A verifier for a language LLL﻿ is an algorithm VV V﻿ such that: L={w ∣ V accepts <w,c> for some string c}L=\{w~|~V \text{ accepts <w,c> for some string }c\}L={w ∣ V accepts <w,c> for some string c}﻿.

A polynomial time verifier is a verifier which runs in polynomial time in the length of www﻿.

If VVV﻿ accepts ⟨w,c⟩\langle w,c\rangle⟨w,c⟩﻿ then ccc﻿ is called certificate that www﻿ is in LLL﻿.

If VVV﻿ runs in polynomial time in the length of www﻿, then the part of ccc﻿ which VVV﻿ explores must have polynomial length in the length of www﻿!

NP

We can now give a definition for NP. Two, even!

The class NP is the class of languages which have polynomial time verifiers.

This means that HAMPATH is in NP - we can check, in polynomial time in the size of the graph, whether a sequence of nodes is a Hamiltonian path.

Alternatively, sometimes it’s easier to think about NP in terms of its namesake:

The class NP is the class of languages which can be decided by non-deterministic Turing machines in polynomial time.

This also shows us that HAMPATH is in NP - the length of each branch in a non-deterministic HAMPATH algorithm is polynomial in length, so a non-deterministic TM will only take polynomial time to decide!

Recall that computation of ND-TMs are trees - usually split into a guessing (branching) and a checking (computing) phase.

As long as the checking phase is polynomial (and you don’t guess in a stupid way), then the ND-TM is polynomial. See how this relates to the verifiers?

Relevance to P

IMPORTANT: P is a subset of NP. We don’t know if this is a strict subset or not - this is the core of P-NP, which is coming up soon. Maybe even in this section!

Problems in NP (or proving a problem is in NP)

If you need some help deciding whether a problem is in NP, here are some examples. They all have a common thread.

HAMPATH is in NP, if you were wondering.

SAT

You know SAT! Boolean satisfiability! Yahoo!

Given a boolean expression, does there exist an assignment of values to variables such that the formula is satisfied?

SAT is in NP.

We can non-deterministically generate all possible assignments of truth values to the variables (guessing), then substitute them over the formula to see if it’s satisfied (checking).

Again, we don’t know if SAT is in P, because of what I said a couple paragraphs ago.

Travelling Salesman

Is there a circular route around a weighted graph which visits each city exactly once, with total distance at most DDD﻿ (where DD D﻿ is in terms of weights)?

TSP(D) is in NP.

We can non-deterministically guess the order in which to visit the cities (guessing), and then check that it’s a valid route with distance at most DDD﻿ (checking).

We don’t know if TSP(D) is in P. You get the idea.

3COL

Is it possible to colour the vertices of a graph with at most 3 colours such that no adjacent vertices share the same colour?

3COL is in NP.

Same deal. Guess colours non-deterministically, check validity of each solution.

P=NP...?

There are lost of NP problems like the above, where the best known algorithm runs in exponential time.

However, no-one’s been able to prove that the only algorithms for these problems are exponential!

We still don’t know whether P=NP or not. We suspect “no”, but it’s still unsolved.
