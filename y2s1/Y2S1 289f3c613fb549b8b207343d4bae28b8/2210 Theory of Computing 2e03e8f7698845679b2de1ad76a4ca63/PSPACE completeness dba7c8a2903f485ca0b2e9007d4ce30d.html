Just like time.

A language BBB﻿ is PSPACE-hard if every language AAA﻿ in PSPACE is polynomial-time reducible to BBB﻿.

BBB﻿ is PSPACE-complete if:

BBB﻿ is in PSPACE

BBB﻿ is PSPACE-hard.

But wait - why not polynomial-space reducible?

We defined NP-complete problems by using polynomial-time reductions because we wanted the reduction to be easy compared to typical problems in NP.

If we used polynomial space reduction here, then an easy solution for the problem we are reducing to would not necessarily give an easy solution to the problem we are reducing from.

True Fully Quantified Boolean Formulae

TQBF is a PSPACE-complete problem, and space’s version of SAT.

The “quantified” part of TQBF introduces new aspects: for all and for each.

The first one is not in TQBF because it’s not true.

The second one is fine.

Back to playing games

You can view deciding TQBF as a deciding a two-player game.

Player A selects values for ∀\forall∀﻿ variables,

Player E selects values for ∃\exist∃﻿ variables,

the order of play is the order of quantifiers in the formula

when all values are chosen, the formula is evaluated - if true, E wins, otherwise A wins.

You can see how A “pushes” the formula out as much as possible, while E is trying to narrow it down as far as possible into truth values. Kinda.

Generalised geography game

Players take turns naming cities - the chosen city has to start with the same letter than ended the previous city. No repetitions are allowed, first player to run out of cities loses.

You can clearly model this as a graph.

This is in PSPACE (see tutorial 10).

A problem in EXPTIME and not in P

In generalised geography, a play takes polynomial time in the size of the graph.

In generalised chess, a play may take exponential time in the size of the board.

Hence, the problem of evaluating a position in generalised chess is EXPTIME-complete but also not in P!
