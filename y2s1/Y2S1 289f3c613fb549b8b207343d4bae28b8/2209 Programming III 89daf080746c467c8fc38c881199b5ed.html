Reminder

A Functor is some box for a type which provides a generalised map: fmap

fmap takes a function from a to b and performs that function on whatever a is inside the box.

class Functor f where
	fmap :: (a -> b) -> f a -> f b

An Applicative is some box for a type which provides a function to put things in the box (pure) and a function to perform application inside the box (<*>), alongside inheriting fmap.

pure takes some a and puts it inside the structure.

<*> takes a function in the box and some value in the box and applies the function to the value, returning the outcome inside the box.

 class Functor f => Applicative f where
	pure :: a -> f a
	(<*>) :: f (a -> b) -> f a -> f b

A Monad is a box which provides the bind operator >>=. It also provides return, for legacy reasons.

>>= takes a value in a box and a function which “takes a pure value and returns some value in a box” and gives you the result of the function acting on the given value in the box.

For example, what if you had a divide function which returned Nothing when it failed and you wanted to apply that to a Maybe Int?

class Applicative m => Monad m where
	return :: a -> m a
	return = pure -- default definition in the class, due to legacy
	(>>=) :: m a -> (a -> m b) -> m b

Seeing Maybe's implementation might help:

>>= :: Maybe a -> (a -> Maybe b) -> Maybe b
mx >>= f = case mx of
	Nothing -> Nothing
	Just x -> f x

Chaining >>= allows you to bind the results of “messy” structured computation to variables and then combine these in a structured return value. This is the essence of do notation.

m1 >>= \x1 ->
m2 >>= \x2 ->
m3 >>= \x3 ->
f x1 x2 x3

-- or,
do x1 <- m1
	 x2 <- m2
   x3 <- m3
	 f x1 x2 x3
