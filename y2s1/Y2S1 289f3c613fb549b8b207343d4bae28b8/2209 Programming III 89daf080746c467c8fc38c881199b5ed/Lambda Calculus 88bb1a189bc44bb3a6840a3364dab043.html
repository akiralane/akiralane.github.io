Lambda Calculus

The lambda calculus can be considered the simplest "interesting" programming language. It's the foundation of all functional programming languages.

In its purest form it contains just two primitives: function application and abstraction.

That means the grammar is simply:

E ::= x | 位x -> E | E E

x is drawn from a set of countably infinite variables.

The 位 terms are called abstractions.

Power

The lambda calculus is surprisingly expressive considering how minimal it is - it's Turing complete, even! Through encodings, you can represent arithmetic, logic, structured data etc. Have a look at an example for pairs:

(a, b) is encoded as 位v -> v a b
fst a  is encoded as e (位x -> 位y -> x)
snd a  is encoded as e (位x -> 位y -> y)

Syntactic conventions

  • Application always associates to the left:
    • a b c means (a b) c
  • Abstraction extends as far to the right as possible:
    • 位x -> 位y -> x y means 位x -> (位y -> x y)
      • not 位x -> (位y -> x) y

Free variables and closed terms

We say that a variable x is bound in the term 位x -> e and that the scope of this binding is the expression e.

Any variable y which is not bound but appears in the scope of an abstraction is called free.

A term which has no free variables is called closed.
