Polymorphism and Classes

Polymorphism

Expressions in Haskell can often have potentially many types. For example, length could have:

[Bool] -> Int
[Int] -> Int
[[Char]] -> Int

This is what the type variables are for:

length :: [a] -> Int

We say that a function is polymorphic if its type contains one or more type variables. Many of the functions defined in the standard prelude are polymorphic.

Not quite polymorphic

Consider the function:

add x y = x + y

It can't accept any type - we need to be able to add x and y.

However, we can add several types - Integer, Float, Double etc.

This is what class constraints are for:

add :: Num a => a -> a -> a

Num is a typeclass. add therefore accepts arguments of any time a that's an instance of the Num class.

Basic typeclasses

Typeclasses are Haskell's way of providing overloaded operations. You can kinda think of it like an interface in OOP - it defines a named type and certain operations that anything implementing that type must support.

There are a number of basic builtin typeclasses. Let's have a look at a few:

Eq and Ord

Eq represents types that support the == and \= comparison operators.

Ord represents types that support order comparisons, including <, >=, min, max, etc.

All of the basic types (plus list and tuples built from them) are instances of Eq and Ord.

Show and Read

Show represents types that support the show function - that is, it can represent itself nicely as a string somehow.

Read represents types that support the read function. This goes the opposite way - parses a string into a value of the type.

Again, all of the basic types are instances of these.
