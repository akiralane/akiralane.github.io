XSLT

XSLT (eXtensible Stylesheet Language Tranformations) is a flexible way of determining how an XML document should be displayed.

It's a step beyond CSS, which just statically assigns properties to elements.

Using XSLT, you can:

Add/remove elements to/from an output file

Re-arrange or sort elements

Perform tests, make decisions

Hide/display elements

Do other things, probably

XSLT makes use of XPath.

Consider this example, which assumes we have an XML document with Pizza nodes which are children of Pizzas:

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

<xsl:template match="/Pizzas">
	<xsl:for-each select="Pizza">
		<li>Pizza!</li>
	</xsl:for-each>
</xsl:template>

</xsl:stylesheet>

This will produce a <li>Pizza!</li> in the final document for every pizza node in the XML.

We can also retrieve both attributes and contents:

<xsl:template match="/Pizzas">
	<html>
	<body>

		<xsl:for-each select="Pizza">
			<p>
				<xsl:value-of select="@name" /> <!-- gets the name attribute of the pizza -->
				has a base of
				<xsl:value-of select="Base" /> <!-- gets the content of the <Base> child node of the pizza -->
			</p>
		</xsl:for-each>

	</body>
	</html>
</xsl:template>
