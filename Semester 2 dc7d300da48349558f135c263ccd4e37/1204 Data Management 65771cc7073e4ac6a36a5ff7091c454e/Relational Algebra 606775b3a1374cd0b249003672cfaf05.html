Relational Algebra

Relational algebra is a procedural query "language" for databases.

Standard set-theoretic operations

In relational algebra, there exist the standard set-theoretic operations:

  • Union, intersection and difference produce relations in the manner you'd expect
  • Cartesian product always produces a 1-dimensional vector; if you cartesian product a relation of (a,b)(a,b)(a,b)﻿ and a relation of ccc﻿, you get outputs of (a,b,c)(a,b,c)(a,b,c)﻿ instead of ((a,b),c)((a,b),c)((a,b),c)﻿

Special relational operations

Renaming

Recall that "columns" have names in relational algebra. When performing a binary operation, you must resolve the names of the output.

This could be done by just having the new relation be unnamed, or by using the unary renaming operator. ρA→B(R)\rho_{A\rarr B}(R)ρA→B​(R)﻿ renames the field named AAA﻿ in RRR﻿ to BBB﻿.

Projection

Projection is a unary operation of the form π<attribute list>(<relation name>)\pi _{<attribute \space list>}(<relation \space name>)π<attribute list>​(<relation name>)﻿.

Intuitively:

  • Applying projection to a relation, it removes all columns whose attributes do not appear in the attribute list.
  • The remaining columns may be re-arranged according to the order in the attribute list.
  • Duplicate rows(?) are also removed.

Instead of attribute names, we can use the position of the column. We denote position by using $ to differentiate from the number as a string.

Selection

Selection is a unary operation of the form σΘ(R)\sigma_\Theta(R)σΘ​(R)﻿ where RRR﻿ is a relation and Θ\ThetaΘ﻿ is a condition that can be applied as a "test" to each row.

Selection returns the subset of RRR﻿ consisting of all rows that satisfy Θ\ThetaΘ﻿.

Θ\ThetaΘ﻿-join

A special query (selection) of the form σΘ(R×S)\sigma_\Theta(R\times S)σΘ​(R×S)﻿ where we combine relations.

For example,

  • Let schema relations FACULTY(name, dpt, salary) and CHAIR(dpt, name).
  • The relation CHAIR doesn't contain salary information, so to find the salaries of department chairs:
    • C_SALARY(dpt, salary) =

      πF.dpt, F.salary(σF.name = C.name∧F.dpt = C.dpt(FACULTY×CHAIR))\pi_{\text{F.dpt, F.salary}}(\sigma_\text{F.name = C.name}\land_\text{F.dpt = C.dpt}(\text{FACULTY}\times\text{CHAIR}))πF.dpt, F.salary​(σF.name = C.name​∧F.dpt = C.dpt​(FACULTY×CHAIR))﻿

R⋈FSR\bowtie_FSR⋈F​S﻿

where FFF﻿ is some condition.

This is NOT to be confused with natural join!!

Equijoins and Natural Joins

An equijoin is a Θ\ThetaΘ﻿-join in which the only predicate operator used is equality (===﻿).

A natural join ⋈\bowtie⋈﻿ is an equijoin over all the common attributes of the joined relations where one occurence of each common attribute is removed from the resulting relation (so there aren't two of each attribute).

The natural join ⋈\bowtie⋈﻿ is the most FAQ in relational databases.

Let A1,…,AkA1,\dots,AkA1,…,Ak﻿ be the common attributes between two relation schema RRR﻿ and SSS﻿.

Then, the natural join of RRR﻿ and SSS﻿ is:

R⋈S=π<list>(σR.A1 = S.A1∧R.Ak = S.Ak(R×S))R\bowtie S = \pi_\text{<list>}(\sigma_\text{R.A1 = S.A1} \land _\text{R.Ak = S.Ak}(R\times S))R⋈S=π<list>​(σR.A1 = S.A1​∧R.Ak = S.Ak​(R×S))﻿

where <list> contains all attributes of R×SR\times SR×S﻿, except for S.A1,… , S.Ak\text{S.A1,$\dots$, S.A$k$}S.A1,…, S.Ak﻿. In other words, duplicate columns are removed.

Outer joins

A (left) outer join ⋊\rtimes⋊﻿ of two relations RRR﻿ and SSS﻿ is a natural join which also includes tuples from RRR﻿ which do not have corresponding tuples in SSS﻿; missing values are set to null via a kinda hacky method.

R⋊SR\rtimes SR⋊S﻿ is

(R⋈S)∪((R−πr1, r2, ... , rn(R⋈S))×{<null,… ,null>})(R \bowtie S) \cup((R-\pi_\text{r1, r2, ... , rn}(R\bowtie S)) \times \{\text{<null,$\dots$,null>}\})(R⋈S)∪((R−πr1, r2, ... , rn​(R⋈S))×{<null,…,null>})﻿

where r1r1r1﻿ etc are attributes in RRR﻿.

Normally there would be a tuple mismatch, but by applying the cartesian product alongside a set of as many nulls as you need, it produces empty values while conforming to the number of attributes.

The left outer join is not the only outer join; the right outer join ⋉\ltimes⋉﻿ includes tuples in SSS﻿ which do not have corresponding values in RRR﻿ (I think?) and the full outer join ⟗⟗⟗﻿ includes all tuples from both relatons.
