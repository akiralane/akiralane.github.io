Just Function Things

In C, the order in which you define functions matters!

float sqrt(float x) {
	/* do some stuff */
	while ( absoluteValue( /*...*/ ) >= epsilon ) {
		/* more stuff */
	}
}

float absoluteValue(float x) {
	if (x < 0) { x = -x; }
	return x;
}

The above code, set up as it is, would get you a conflicting types error.

This is because the compiler finds the absoluteValue call in the while loop, sees that it hasn't been defined yet, and assumes that it returns an int. Then, when the definition is provided down the code, it sees that you've named it a float instead and freaks out.

The solution to this is to declare (not necessarily define...!) the function before it is called. This gives the compiler the signature for the method right at the start, avoiding any such errors.

#include <stdio.h>

float absoluteValue(float x);
float sqrt(float x);

/* define the functions (and the rest of the code) down here... 
...
... 
*/

Pass-by-value vs pass-by-reference

This is pretty much the same as you've known it. In C, if you give something as an argument, its value is copied into the parameter variable - any changes to that variable will not take effect outside of the function scope.

You can, however, take a pointer as an argument. This will allow you to mutate the value at the pointer from inside the function.

Similarly, you can define functions which return pointers.

Structures

Structures are a tool that allow you to group together elements of different types. The general form is:

struct name {
	type member1;
	type member2;
	type member3;
	/* ... */
}

As you can see, this gives you a sort of data template.

After defining the structure, you can declare variables using it as follows:

struct time { int second; int minute; int hour; }

/* ... */

struct time meeting;     // meeting is declared with members second, minute and hour
struct time currentTime; // etc.

meeting.hour = 14; // these are accessed in the way you'd expect
currentTime.sec = 38;

// you can also initialise in declaration
struct time meeting_2 = {38, 23, 14};
struct time meeting_3 = {38, 23}; // hour is left uninitialised
struct time meeting_4 = {.hour = 14, .sec = 38}; // minute is left uninitialised

Structs and pointers

Obviously, a pointer can point to a struct. You'll need to use brackets to access members though, since . binds tighter than *.

Alternatively, there is also the special structure pointer operator ->, which isn't convoluted at all.

(*pointer).member = 200;
// alternatively,
pointer->member = 200;

Type alias

The keyword typedef creates an alias which can be used anywhere in place of a (possibly complex) type name.

That alias is exactly equivalent to the type which it is aliasing.

typedef int* pointer;

// the following are identical;
pointer x;
int* x;

This can make code significantly more concise and readable, but only really in certain situations.

Using typedef with structures

There are two ways of using typedef with struct:

// either...
struct time { int second; int minute; int hour; }
typedef struct time TIME_TYPE;

// ...or anyonymously inline:
typedef struct { int second; int minute; int hour; } TIME_TYPE;

Using typedef and structs with functions

Functions can have structures as arguments and return structures, as you might expect.

Using the time struct that we've been using as an example:

struct time update(struct time now) { /* ... */ }
// or, using our alias,
TIME_TYPE update (TIME_TYPE now) { /* ... */ }
