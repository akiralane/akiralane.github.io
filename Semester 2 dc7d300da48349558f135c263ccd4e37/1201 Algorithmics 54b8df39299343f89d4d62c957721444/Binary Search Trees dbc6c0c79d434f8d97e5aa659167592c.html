Binary Search Trees

Trees are one of the major ways of structuring data in order to allow efficient access.

Mathmatically, a tree is an acyclic undirected graph. That is, it consists of nodes joined by undirected edges which cannot have any "cycles" in (there is only one route to each node...?). This means that a single node can also count as a tree.

We often impose an ordering on the nodes - this is known as a rooted tree.

The top node is the root node, while all nodes with no children are leaves.

We can think of a tree as being made up of subtrees (plus the root). Hence, it is useful to label different "levels" of the tree; we take the level of a node as being its distance from the root. The height of a tree is the number of levels it has.

Binary Search Tree

We will concentrate on one of the most important binary trees: the binary search tree. The binary search tree keeps the elements ordered.

We can define such a tree recursively:

  1. Each element in the left subtree is less than the root element
  1. Each element in the right subtree is greater than the root element
  1. Both left and right subtrees are themselves binary search trees.

The structure of the tree therefore depends on the order in which we add elements to it.

Searching a binary search tree is easy.

  1. Start at the root.
  1. Compare with element.
    1. If less, it has to be in the left tree, so go left.
    1. If greater, it has to be in the right tree, so go right.
    1. If equal, we have found the element.

The number of comparisons necessary to find an element in a binary tree depends on the level of the node in the tree.

The worst case number of comparisons is therefore the height of the tree.

Inorder Binary Tree Walk

We can print out all elements in a binary search tree in sorted order using a very simple recursive algorithm. It takes advantage of the fact that the binary tree is made of subtrees which are themselves binary trees:

def walk(node):
	if (not node is None):
		walk(node.left)
		print(node.element)
		walk(node.right)

Implementing a Set using Binary Search Trees

Sets are a very useful data type; one of the most efficient ways of implementing a set is through a binary search tree.

Ironically, because order doesn't matter here, we can order the elements how we like - hence, we sort it in a binary search tree. This allows us to search through it rapidly.

Furthermore, binary search trees cannot have duplicate elements, which is perfect for a set.

Tree Iterator

To iterate through the elements from the start we start at the left-most branch and node.

To find the successor of the current node (in the ordering of elements), we follow two rules:

  1. If there exists a right child, move right once and move far left as possible
  1. Else, go up to the left as far as possible and then move up right once.
