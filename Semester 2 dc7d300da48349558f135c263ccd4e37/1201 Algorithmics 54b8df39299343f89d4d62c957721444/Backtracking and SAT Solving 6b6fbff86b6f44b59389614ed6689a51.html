Backtracking is used to find feasible solutions in large state spaces.

Fundamentally, it is used in search problems - many real world applications involve taking a series of actions to manipulate the state of a system.

It works until either:

  1. a feasible solution is found (and we can finish)
  1. no feasible solution is possible (and we backtrack).

Giving up on states which would lead to other infeasible states eliminates the need to consider the other said states (whereas a brute force solution considers each individually; it would have no idea about each state until it checked it).

We want to put four queens on a 4x4 chess board such that none of them threaten each other.

We know that no two queens can be on the same row, so we will move through the board row-by-row (or index by index in our vector).

To start, we consider the first square. This obviously doesn't cause any conflicts in our constraints, so we fill it with a queen to get (0,0,0,a)(0,0,0,a)(0,0,0,a)Ôªø.

Proceeding to the next row, we place a queen in the first position that isn't under attack; this gives us (0,0,c,a)(0,0,c,a)(0,0,c,a)Ôªø

We check the next row. However, we can't place a queen in any position here without leaving it under attack. This is where we backtrack, and find an alternate valid position for the second row.

You see how this goes.

SAT Solving using Backtracking

The Boolean satisfiability problem asks whether a given Boolean formula is satisfiable - that is, whether there exists some assignment of truth values that makes the formula evaluate to true.

(The SAT problem was the first example of an NP-Complete problem!)

Just like in Linear Programming, there are very many problems we can encode as a SAT problem.

For example, problems in circuit design can often be cast as SAT problems: consider a model where an output from a wire corresponds to a Boolean formula whose variable correspond to the input wires.

You might model a multiplier circuit as a Boolean formula, which allows you to ask if there are (binary number) inputs to the circuit that result in a given number in the output.

This is equivalent to asking if said number is prime or not!

DPLL

The Davis-Putnam-Logemann-Loveland algorithm is a backtracking-based algorithm for solving the SAT problem. It serves as the backbone of modern SAT solvers and has major practical applications in hardware verification. It's a pretty impactful algorithm!

Although it can take exponential time in the worst case, it can often find solutions to SAT relatively quickly.

DPLL works by:

  1. Converting the Boolean formula into Conjunctive Normal Form
  1. Applying simplification rules
  1. Splitting and backtracking
