Matching

A matching is a set of edges without any common vertices. That is, it's an independent edge set.

A set of vertices is "covered" by a matching MM M﻿ is every vertex in the set is incident with an edge in MMM﻿.

MMM﻿ is a maximal matching when M+eM+eM+e﻿ is not a matching, where eee﻿ is any unmatched vertex. That is, you can't add any more edges without having edges share vertices.

Alternatively but equally, MMM﻿ is maximal if it's not a subset of any other matching.

A maxiMUM (cardinality) matching is a matching that contains the largest possible number of edges.

Some remarks:

Neither maximal nor maximum matching are necessarily unique.

MMM﻿ is maximum   ⟹  \implies⟹﻿MMM﻿ is maximal, but not the other way round.

From maximal to maximum

Consider that the "symmetric difference" of some MM M﻿ is a matching with one more (/one less?) edge than that of MMM﻿. This is acheived by alternating the edges.

Given a graph G=(V,E)G=(V,E)G=(V,E)﻿ and a matching MMM﻿, a path PPP﻿ is called an augmenting path if:

  • The two end points of PPP﻿ are unmatched by MMM﻿.
  • The edges of PPP﻿ alternate between edges ∈M\in M∈M﻿ and edges ∉M\notin M∈/M﻿.
Berge's Theorem A matching MMM﻿ is maximum iff it has no augmenting path.

We can use this idea to find the maximum matching.

let M = {}
while there exists an augmenting path P w.r.t M
	flip the edges in P - perform the symmetric difference operation
return M

So how do we find an augmenting path? Graph traversal.

Given a matching MMM﻿, we start with an unmatched vertex v0v_0v0​﻿ as the root and run a depth first search.

However, this doesn't guarantee finding an augmenting path.

These cases happen because of odd cycles. In an odd cycle, there will be two unmatched edges that share a vertex.

Bipartite Graphs

Recall that a graph is bipartite iff it is 2-colourable. (You can find a bipartite graph visually if you can colour it with two colours, where no two adjacent nodes have the same colour). That is, it's bipartite iff it contains no odd cycles.

That means our maximum matching algorithm will find a maximum matching.

A bipartite graph satisfies the property tr(Ak)=0tr(A^k)=0tr(Ak)=0﻿ for k=3,5,6,…,2⌈n2⌉−1k=3,5,6,\dots,2\lceil{n\over2}\rceil-1k=3,5,6,…,2⌈2n​⌉−1﻿.

if G is a bipartite graph
	M = {}
	start DFS at an unmatched vertex
	while there exists an augmenting path P w.r.t M
		flip the edges in P
	return M

Blossoms

Given a non-bipartite graph GGG﻿ and a matching MMM﻿, a blossom is an odd cycle where one of its vertices is such that there exists an alternating path of even length from the vertex to an unmatched vertex. The alternating path is named the stem.

You can shrink a blossom down to effectively represent a single node (and vice versa):

This gives us a new recursive idea for a maximum matching algorithm:

# Edmonds' Blossom Algorithm
M := {}
start graph traversal at an unmatched vertex
if there exists an augmenting path P wrt M
	flip the edges in P
else,
	find a blossom B and shrink it
	fina a maximum matching M/B in G/B         # obtain an equivalent smaller problem
	expand B, obtain a maximum matching M in G
return M

The idea is that MMM﻿ is a maximum matching in GGG﻿ iff M/BM/BM/B﻿ is a maximum matching in G/BG/BG/B﻿.

This gives us a time complexity of O(mn2)O(mn^2)O(mn2)﻿ or O(n4)O(n^4)O(n4)﻿.

Algorithm Design thoughts

  • Brute force
    • Usually prohibitively expensive.
  • Greedy
    • Short-sighted; sometimes local optimum is global optimum, but sometimes it isn't.
  • Optimal algorithms
    • Need insights and rigor proofs
  • Approximation algorithms
    • Competitive analysis
  • Dynamic programming (coming up)
