Approximation Algorithms

Approximation algorithms are algorithms that find approximate solutions to optimisation problems (in particular, NP-hard problems) with provable guarantees about their distance to the optimal solution.

A little complexity theory

P is a complexity class that contains all decision problems (yes/no problems) which can be solved by a deterministic Turing machine in polynomial time.

NP (non-deterministic polynomial-time) is a complexity class that contains all decision problems for which the answer can be verified by a deterministic Turing machine in polynomial time.

NP-Complete means that the problem is at least as hard as any problem in NP. This is important because it has been shown that any problem in NP can be transformed into another problem in NP-Complete; a solution to any one NP-Complete problem is a solution to all NP problems. (see the section below for a more formal definition)

Polynomial time reduction

A polynomial time reduction B≤pAB \leq_p AB≤p​A﻿ exists where:

  • The steps required to transform any input of the problem BBB﻿ to an input of problem AAA﻿ is polynomial
  • The number of times the subroutine is called is polynomial

A problem AAA﻿ is NP-Complete if:

  • AAA﻿ is in the NP complexity class
  • Every BBB﻿ in NP has a polynomial time reduction to AAA﻿.

Hence, if we could solve some AAA﻿ quickly, we could solve every problem in NP quickly, since we could effectively turn any problem in NP into some combination of AAA﻿s.

It is most commonly believed that P≠\neq=﻿NP.

Bin Packing

Given an infinite set of fixed-size bins and a number of items of differing sizes, we want to pack all the items into the fewest bins possible.

Next Fit

Next fit is a fairly simple algorithm:

if item i fits into bin j:
	pack it, i++
else
	j++

That is, just pile them in and move to the next one when full.

It runs in O(n)O(n)O(n)﻿ (where n is the number of items, not bins), but how good is it?

Let s be the number of non-empty bins.
