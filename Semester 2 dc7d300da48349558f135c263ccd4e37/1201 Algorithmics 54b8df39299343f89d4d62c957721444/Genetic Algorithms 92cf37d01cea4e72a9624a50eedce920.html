Some Terminology

  • Individual: any possible solution
  • Population: a set of individuals
  • Fitness: the objective function we are trying to optimise

    (every individual can be evaluated according to this function)

  • Chromosome: representation of a solution

    (often take the form of strings or vectors)

  • Gene: a position (or set of positions) in a chromosome
  • Allele: the possible values in a gene

For example, the chromosome of an individual with alleles A, B, ..., G could be:

ABBCDAFAGBBAAFGFCBCA

Application of Genetic Algorithms: Fuzz Testing

The main idea behind fuzzing is to observe a program's behaviour on inputs that are not quite valid. There are two common approaches:

Mutation based fuzzing, where you start with a well-formed input as a seed and mutate it to generate new inputs.

Generation-based fuzzing, where you use the grammar of the input format in order to generate inputs from scratch.

AFL - American Fuzzy Lop

AFL instruments a program at compile time (so that it notifies AFL whenever a particular branch is taken), and uses a genetic algorithm to find inputs that trigger new internal states in the target.

Fitness of the individuals (the inputs) depends on their ability to "cover" new branches in the target; fit inputs are those that cause the target to behave differently in order to test new parts of the program

It's highly effective and has historically discovered many vulnerabilities.

Heuristic Conclusions: which is best?

Obviously, the best one depends on the application.

Descent/hillclimb/neighbourhood search is very fast, but only finds local optima; it's a good starting place.

Simulated annealing and genetic algorithms are both slow, but often find good solutions.

The best algorithms tend to be special-purpose algorithms designs per-problem.
