AVL Trees

An AVL tree is a binary search tree in which:

  1. the heights of the left and right subtree differ by 1 level at the most
  1. the left and right subtrees are themselves AVL trees

This guarantees that the worst case AVL tree has logarithmic depth.

Implementing AVL trees

To implement an AVL tree, we need to include additional information at each node indicating the "balance" of the subtrees.

balanceFactor = (height of left subtree) - (height of right subtree)

We want to maintain this when implementing an AVL tree. This means we need to:

  1. rebalance the tree using rotations if the balanceFactor goes outside of the defined bounds

Deletion

AVL deletions are similar to AVL insertions, in that you need to update the balance factors.

One difference is that after performing a single rotation, the tree might still not be within the AVL criteria so higher levels would need to be examined.
