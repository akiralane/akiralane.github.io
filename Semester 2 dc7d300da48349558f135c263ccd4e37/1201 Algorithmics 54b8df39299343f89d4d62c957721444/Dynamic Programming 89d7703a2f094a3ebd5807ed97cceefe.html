Dynamic Programming

Dynamic programming is solving sub-problems, remembering the solutions and reusing those solutions.

Remember the When Definitely Not to Recurse section in Recursion?

Does DP always work?

Cycles matter! We could end up with values which are defined in terms of themselves from a DP perspective, which isn't useful at all when finding a concrete solution.

Directed acyclic graphs are fine.

Still, not all problems can be split neatly into subproblems - and, very often, it needs a careful design.
