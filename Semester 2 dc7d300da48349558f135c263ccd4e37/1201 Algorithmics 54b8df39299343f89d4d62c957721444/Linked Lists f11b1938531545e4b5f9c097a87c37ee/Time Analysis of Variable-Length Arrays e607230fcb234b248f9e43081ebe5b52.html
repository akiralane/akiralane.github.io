Time Analysis of Variable-Length Arrays

When an array is at full capacity we have to copy all elements to a new "doubled" array in order to add more.

If we have an inital capacity of 10 and add 100 more, doubling the array size whenever required, then the number of operations needed is:

  • array inserts: 100
  • element copies: 10 + 20 + 40 + 80 (because it has to copy everything again when the doubled array becomes full)
  • array creations: 4

If we perform NNN﻿ adds with an initial capacity of nnn﻿, we have to perform mmm﻿ entire array copies.

The number of adds NNN﻿ is bounded by how many full copies we have to do - rather, the number of full copies mmm﻿ is determined by the number of adds, because the array has to double when we try to add to a full array:

2m−1n<N≤2mn2^{m-1}n<N\leq2^{m}n2m−1n<N≤2mn﻿
