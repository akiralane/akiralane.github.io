Recursion

Recursion is the strategy of solving a problem by breaking it down into multiple smaller but otherwise identical problems until we reach a trivial case that we can solve by other means.

A recursive definition is made of two parts:

  • Base/Boundary Case: where solving the problem is trivial,
  • Recursive Clause: a self-referential part driving the problem towards the base case.

This is similar to induction - the two are very closely related.

Greatest Common Divisor

One of the most famous algorithms is Euclid's algorithm for calculating the greatest common divisor.

Euclid's algorithm uses the following:

gcd(A,B)=gcd(B,Amod  B)gcd(A,B)=gcd(B,A\mod B)gcd(A,B)=gcd(B,AmodB)﻿

gcd(A,0)=Agcd(A,0)=Agcd(A,0)=A﻿

Thus, gcd(70,25)=gcd(25,20)=gcd(20,5)=gcd(5,0)=5gcd(70,25)=gcd(25,20)=gcd(20,5)=gcd(5,0)=5gcd(70,25)=gcd(25,20)=gcd(20,5)=gcd(5,0)=5﻿.

This is trivial to implement with recursion:

public static long gcd(long a, long b) {
	if (b == 0) {
		return a;
	} else {
		return gcd(b, a % b);
	}
}

Writing Recursive Programs

You need to make sure that you catch the base case before you recurse.

Ensure that your recursive calls use a "smaller" problem AND assume that you can solve the "smaller" problem.

Why Use Recursion At All?

Both factorial and gcd could be written without using recursion and they would probably end up being faster.

However, the cost of the additional function calls is often insignificant. It becomes considerably more complex to read and write certain algorithms if you don't use recursion.
