Minimum Spanning Tree

Given a connected, undirected, weighted graph G=(V,E)G=(V,E)G=(V,E)﻿, we want to find a subgraph T=(V,E′)T=(V,E')T=(V,E′)﻿ of GGG﻿ such that:

  • TTT﻿ spans all vertices of GGG﻿ - we visit every node.
  • The total weight of the edges in TTT﻿ is minimised.

Such a tree is called a minimum spanning tree.

Given an existing set of edges AAA﻿, an edge (u,v)∉A(u,v)\notin A(u,v)∈/A﻿ is a safe edge if it has the minimum weight and adding it to AAA﻿ does not destroy the tree-structure of AAA﻿.

Algorithms to grow a Spanning Tree

GENERIC_MST(graph, weights)
	T := {}
	while T does not form a spanning tree do
		find an edge (u,v) which is a safe edge for T
		T = T union { (u,v) }
	return T

The big question here is how to find the next safe edge.

Krukal's

In Kruskal's algorithm, the set TTT﻿ is a forest.

The next safe edge is the one with the least weight and does not form a cycle with edges in TTT﻿.

This means that the next safe edge does not necessarily have to connect to the existing edges.

// sort the edges in E in increasing order according to weights
T := {}
while E is not empty do
	take an edge "e" which is the shortest edge in E
	delete e from E
	if e does not form a cycle with edges in T
		then add e to T
return E

How do we decide (in the if) whether a new edge forms a cycle with TTT﻿?

Idea: if the new edge forms a cycle, the edge has to connect two nodes representing the same subtree.

Thus, we need to (quickly) find which subtree the two vertices of an edge belong to.

For this, we use the union-find data structure:

Initially, all nodes are assigned to their own separate subtrees.

When two subtrees are combined by and edge, we perform the union of the two subtrees.

When all of the subtrees have been unioned together, we know we're done and have formed a tree which holds every node.

Here's a walkthrough of how this works for the above graph:

We're looking through edges in order sorted by their weight:

X     | {1} {2} {3} {4} {5} {6} {7}
{1,2} | {1,2} {3} {4} {5} {6} {7}
{2,3} | {1,2,3} {4} {5} {6} {7}
{4,5} | {1,2,3} {4,5} {6} {7}
{6,7} | {1,2,3} {4,5} {6,7}
{1,4} | {1,2,3,4,5} {6,7}
{2,5} | REJECTED: we try to connect 2 -> 5, but these nodes are in the same subtree. This would create a cycle.
{4,7} | {1,2,3,4,5,6,7}
== DONE: all nodes are in one subtree ==

Prim's

In Prim's algorithm, the set TTT﻿ is a single tree.

The next safe edge is the one with the least weight and is connected to TTT﻿ without forming a cycle.

This means that the next safe edge is always connected to the existing tree.

// sort the edges in E in increasing order according to weights

choose some vector v in V
L := {v}
R := V - {v}
T := {}
while R is not empty do
	take an edge "e" that has the least weight amongst edges crossing L and R
	add e to T
	remove the node which e comes from from R and add it to L
return T
