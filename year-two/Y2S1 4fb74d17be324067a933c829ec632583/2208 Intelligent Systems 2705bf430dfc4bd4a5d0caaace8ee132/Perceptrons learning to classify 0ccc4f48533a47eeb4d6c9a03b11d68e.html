Intro

Our goal:

Given data DDD﻿, construct a model fff﻿ such that the distance between model output and real "output" is small.

We take xnx_nxn​﻿ as an input data point and yny_nyn​﻿ as an output data point. yn^\hat{y_n}yn​^​﻿ is the output predicted by fff﻿.

If yny_nyn​﻿ is discrete (like +1/-1 or cat/dog/bird) then fff﻿ is a classification model.

If it's continuous, fff﻿ is a regression model.

The mismatch between fff﻿'s prediction and the 'real output' (or given labels) is quantified as the loss function. "Learning" is then the development of the model through minimising the loss LLL﻿.

Here's the above written in notation:

Data: D:={(xn, yn)}, n=1,…,ND:=\{(x_n,~y_n)\},~n=1,\dots,ND:={(xn​, yn​)}, n=1,…,N﻿

Model prediction: yn^=f(xn)\hat{y_n}=f(x_n)yn​^​=f(xn​)﻿

Loss: L=∑n=1N Δ(yn^, yn)L=\sum_{n=1}^{N}~\Delta(\hat{y_n},~y_n)L=∑n=1N​ Δ(yn​^​, yn​)﻿

Feature space

We represent our data in some space, so that we can define our decision boundary as a (hyper)plane in that space:

The decision boundary is determined by the weight vector www﻿ (our green line from the diagram before). For a two-dimensional space like we have above, we have the decision boundary:

w⊺x=w0(1)+w1x1+w2x2=0w^\intercal x = w_0(1) + w_1x_1 +w_2x_2=0w⊺x=w0​(1)+w1​x1​+w2​x2​=0﻿

Notice that we've actually picked a value for x0x_0x0​﻿ here: since we're taking the dot product of the two vectors which would otherwise be mismatched, we fill out the additional space in xxx﻿ by setting it to 111﻿.

x=(1x1x2)x=\begin{pmatrix} 1 \\ x_1\\x_2 \end{pmatrix}x=⎝⎛​1x1​x2​​⎠⎞​﻿ but w=(w0w1w2)w=\begin{pmatrix}w_0\\w_1\\w_2\\\end{pmatrix}w=⎝⎛​w0​w1​w2​​⎠⎞​﻿

This is the same for all dimensions: x0x_0x0​﻿ should be 1, just to make our maths nicer.

You can see here that we're actually just looking at the equation of a line! It's just done in a generalised manner so that we can apply it to higher dimensions.

w⊺xw^\intercal xw⊺x﻿ partitions the plane into positive and negative regions for any xxx﻿! This is why it's our decision boundary for the classification model.

Perceptron algorithm: update step

We've noticed that our model is wrong for training vector xxx﻿. What now?

We use its location to alter the weight vector and create a new decision boundary:

wt→wt+1=wt±ηxw^t \rarr w^{t+1} = w^t \pm \eta xwt→wt+1=wt±ηx﻿

We can extract www﻿ from the equation (which is why we rewrote it in that form), and we know the vector of our incorrectly predicted point:

w=(0.60.32.0), xwrong=(12.70.2)w=\begin{pmatrix}0.6\\0.3\\2.0\end{pmatrix},~ x_{wrong}=\begin{pmatrix}1\\2.7\\0.2\end{pmatrix}w=⎝⎛​0.60.32.0​⎠⎞​, xwrong​=⎝⎛​12.70.2​⎠⎞​﻿

So we adjust the weight vector:

w−ηx=(0.60.32.0)−0.3(12.70.2)=(0.3−0.51.9)w-\eta x =\begin{pmatrix}0.6\\0.3\\2.0\end{pmatrix}-0.3\begin{pmatrix}1\\2.7\\0.2\end{pmatrix}=\begin{pmatrix}0.3\\-0.5\\1.9\end{pmatrix}w−ηx=⎝⎛​0.60.32.0​⎠⎞​−0.3⎝⎛​12.70.2​⎠⎞​=⎝⎛​0.3−0.51.9​⎠⎞​﻿

We picked an arbitrary learning rate of 0.3.

This should now at least be closer to correctly classifying the value. If our learning rate is small, this could take several steps.

How does this fit into the general story?

  • The perceptron algorithm is based on updates driven by misclassification.
  • If there aren't any errors left, then there are no more iterations. One it's done with the training data, it's done.
  • We only converge on a solution in a perceptron if the data is linearly separable. It's hard to tell if the perceptron will converge for any given set of data.
  • Even if the data is linearly separable, there could be multiple solutions for www﻿.
