Writing Functions: Guards

When something goes wrong, we can use error :: String -> a to stop execution and report the error, in the same way as Rust's panic. I assume we'll look at more precise methods later, but we can use error a lot in these simple examples.

Guarded Equations

Basic if-then-else conditional statements can get very messy, especially when they're nested.

A far preferable solution is to use guarded equations. These allow you to define the function separately depending on the case:

abs :: (Ord a, Num a) => a -> a
abs n | n >= 0    = n
    n | otherwise = n

Guards are evaluated top-down. The first guard which passes is the one which gets to decide the definition.

otherwise is actually just an alias for True - that is, this guard will always pass.

Here's another example:

quadroots :: Float -> Float -> Float -> String
quadroots a b c | a == 0            = error "not quadratic"
								| discriminant == 0 = "the root is " ++ show centre
                | otherwise         = "roots are " ++ show (centre + offset)
                                           ++ ", " ++ show (centre - offset)
	where discriminant = (b*b)-4(a*c)
			  centre       = -b/(2*a)
        offset       = sqrt(discriminant)/(2*a)

show does what you think it does.

Local definitions using where can hugely improve readability. You can also use let to define things locally, but I don't think it looks as nice for most cases:

let name = localExpr in mainExpr
-- you can see how this might get hard to read with many nested lets
