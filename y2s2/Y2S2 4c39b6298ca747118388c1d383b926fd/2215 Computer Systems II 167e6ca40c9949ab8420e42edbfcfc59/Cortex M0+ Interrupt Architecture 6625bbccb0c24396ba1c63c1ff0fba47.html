Cortex M0+

Nested Vectored Interrupt Controller (NVIC)

The RP2040 has two Cortex M0+ cores. Each core has its own Nested Vectored Interrupt Controller.

All external interrupts go to all NVICs. The NVIC can then decide what to do with the IRQ.

  • The NVIC has 26 external interrupt inputs with 4 priority levels, controlled in software.
  • It has very low latency.
  • It allows for tail-chaining.
    • This allows the processor to go straight from an interrupt to another interrupt without restoring the state of normal operation.
    • Without this, we would have to incur the penalty of context-switching overhead if we wanted to process back-to-back interrupts.
  • It allows late-arriving interrupts.
    • If a high-priority interrupt comes in late and the processor has already started servicing a lower priority interrupt, we can stop the lower-priority one.

Installing an ISR

Shared ISR:

For multiple sources which share an IRQ number.

All installed ISRs will be called in order of priority.

Exclusive ISR:

Used if only one ISR will be set on a core for a certain source.
