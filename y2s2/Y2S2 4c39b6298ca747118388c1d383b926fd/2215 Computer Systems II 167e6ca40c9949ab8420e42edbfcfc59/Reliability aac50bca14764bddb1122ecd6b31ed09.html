Reliability

Even if it is, things can go wrong anyway - arbitrary bit flips or signal interference etc.

Redundancy

In order to protect against such faults, we need redundancy.

  • Redundant hardware
    • Protects against random hardware faults
  • Different implementations
    • Protects against systematic faults
  • Redundant software states
    • For example, error correction.

Redundant units need to communicate to be useful, and this communication can itself fail.

Integrity checks

Is anything wrong?

  • Parity checks
    • Typically available in hardware

ECC

If we know which bit is wrong, we can simply flip it back. With enough redundancy, some level of errors can be fully corrected.

If you compress, then add ECC, then failures will look less like a smooth degradation and more like a sudden dropoff.

This makes problem recognition easier.

Redundancy tradeoffs

Redundancy increases the state space and complexity of the system.

Only a secure system can be guaranteed to be reliable. However, increasing reliability often makes security harder itself.
