We say a state yyy﻿ can simulate a state xxx﻿ if:

Whenever xxx﻿ can do some action (becoming x′x'x′﻿ in the process), yyy﻿ can reply with the same action and become y′y'y′﻿.

Note that by the “same action”, it does have to have the same label. This is partially because we said actions can be effectful, so the processes must have the same effects.

The simulation game

We are playing against a demon. The game starts at position (x,y)(x,y)(x,y)﻿.

  1. The demon picks a move xundefinedax′x\xrightarrow{a}x'xa​x′﻿
  1. We much choose a y′y'y′﻿ such that yundefineday′y\xrightarrow{a}y'ya​y′﻿
  1. The game goes back to step 1, changing the position to (x′,y′)(x',y')(x′,y′)﻿.

If at any point a player cannot make a move, that player loses.

If the game goes on forever, then we win.

If the demon has a winning strategy, then yyy﻿ does not simulate xxx﻿. If we have a winning strategy, then yyy﻿ does simulate xxx﻿.

Simulation equivalence

States xxx﻿ and yyy﻿ are said to be simulation equivalent if both x≤yx\leq yx≤y﻿ and y≤xy\leq xy≤x﻿.

This means that to check that two states are simulation equivalent, we usually construct two simulations.

Simulation equivalence and trace equivalence

Equally, if xxx﻿ and yyy﻿ are not trace equivalent, then they cannot be simulation equivalent.

The set of all simulation equivalent state pairs is a strict subset of the set of all trace equivalent pairs.

Simulation equivalence is a finer (or stronger) equivalence than trace equivalence.
