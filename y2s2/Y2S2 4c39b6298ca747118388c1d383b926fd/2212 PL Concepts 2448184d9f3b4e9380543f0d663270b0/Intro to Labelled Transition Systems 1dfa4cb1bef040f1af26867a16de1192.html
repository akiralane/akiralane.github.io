Is it enough?

Concurrent systems are hard to verify and test using typical methods - there’s a massive aspect of non-determinism inherent to them (as reflected even in calculi which represent them!).

So, we need a new method of reasoning.

We use a tree-like structure called a Labelled Transition System in order to capture what can be observed about programs. They can express non-determinism as part of their structure.

Formally, we describe it LTS=(X,Σ,L)LTS=(X,\Sigma, L)LTS=(X,Σ,L)﻿ where:

  • XXX﻿ is a set of states
  • Σ\SigmaΣ﻿ is an alphabet of actions
  • L⊆X×Σ×XL\sube X\times\Sigma\times XL⊆X×Σ×X﻿ - a mapping between states via actions.

LTSs vs FSM

They’re very similar, but not the same.

Importantly, the states of an LTS can be infinite! Further, LTSs typically don’t have initial and final states.

Types of non-determinism

Internal - “the machine chooses”

The non-determinism is resolved by the scheduler.

External - “the environment chooses”

For example, interactive systems. The combination of a vending machine and user can be thought of as a concurrent system!

Trace equivalence

One attempt to describe the specification is as a set of traces (a language...?).
