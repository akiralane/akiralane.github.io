Variance

We’ve seen that - in the type formers for pairs and sums - there’s a relationship between the subtyping on the types of substructure and the subtyping of the structure itself.

For example,

if T<:UT<:UT<:U﻿ and V<:VV<:VV<:V﻿

then T×V<:U×VT\times V<: U \times VT×V<:U×V﻿

Notice that the ordering between TTT﻿ and UUU﻿ is somehow preserved in the latter subtyping relation.

This preservation of order has a name - we say that the pair type former is a covariant type constructor.

There are type formers which don’t behave like this.

For example, what if we have a type former called Foo which, given a type T, resulted in a new type Foo T.

If subtyping for Foo is such that:

if T<:UT<:UT<:U﻿

then Foo U<:Foo T\text{Foo } U <: \text{Foo }TFoo U<:Foo T﻿

then we call Foo a contravariant type former.

Arrays

The situation for arrays is a little more complicated, since we don’t build elements of the type using data constructors.

Elements in the array can be modified.

That is, the array must both consume data that it’s given (write) and produce data when requested (read).
