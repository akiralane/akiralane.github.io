Type environments - let

If we wrote a naive rule for the let expression in Toy, it might look something like this:

But what if we were to try and apply this rule to let ( x : Int ) = 10 in x + 1?

How do we guarantee that the yellow highlighted bit holds? We don’t know anything about x.

We can’t - the rule is a broken one.

So, we need some mechanism to represent assumptions about types of the free variables in an expression. We call these assumptions Typing Environments, represented by Γ\GammaΓ﻿.

Formally, a type environment Γ\GammaΓ﻿ is a mapping from variables to types.

We write this out comma-separated:

x:Int, y:Bool, z:Int, …x:\text{Int}, ~y:\text{Bool},~z:\text{Int},~\dotsx:Int, y:Bool, z:Int, …﻿
