Domain names

Domain names are delegated from ICANN through the Top Level Domain (TLD) registrars.

This allows sites and organisations to be delegated the authority to maintain information about the domains they have registered.

Looking up data in the DNS

The most common DNS lookup is host name to IP.

The DNS gives you the A record(s) (IPv4) or AAAA record(s) (IPv6) associated with the given host name.

This means that clients have to know the IP address(es) of a local DNS server - known as a resolver - that they can send queries to.

Resolvers are local DNS servers that hosts can send queries to. They ‚Äúresolve‚Äù these queries and send them back.

Clients are configured to use one or more local DNS resolvers.

ECS has internal-facing DNS resolvers.

Queries are usually UDP. They get sent to name servers until there‚Äôs a response.

DNS infrastructure

A campus, for example, will run its own DNS servers.

But then how do other systems outside the campus know how to find our DNS servers in order to send queries?

Specifically, how can an external DNS resolver find the IP of our campus DNS servers?

This is where the hierarchy of DNS comes in.

ICANN manages a number of ‚Äúroot‚Äù DNS servers.

The root servers know where the DNS servers for each TLD are.

(that is, for .org, .uk, .com, .net etc.)

And so on; each server knows where the name servers for their delegated subdomains are.

A DNS resolver can therefore be configured with IP addresses for the root servers and then choose one from which to iteratively ‚Äúwalk‚Äù the hierarchy from the root in order to resolve a lookup.

Recursive DNS lookup

Requests go to a resolver. If it doesn‚Äôt know how to resolve a request:

It first asks a root DNS server, which it already knows the IP of.

The root will then tell it to try another DNS server lower down in the hierarchy.

This continues until the name server it wanted is found.

Anycast

Anycast is an exception to the idea of every device having a unique IP address.

It allows a client to reach the nearest instance of a service.

This implies that you can advertise the same IP (or IP block) at multiple points on the internet.

Routers learn of the nearest instance topologically via the routing system.

This means you‚Äôll access a different instance depending on where you are.
