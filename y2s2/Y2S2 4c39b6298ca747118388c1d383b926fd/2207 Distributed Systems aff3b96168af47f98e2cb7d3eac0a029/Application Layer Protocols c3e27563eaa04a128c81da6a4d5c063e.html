DHCP (Dynamic host configuration protocol)

We looked at DHCP briefly before in the context of IPv4, I think - it’s how systems get a dynamic IP address.

Telnet

It’s a simple unencrypted terminal emulation protocol.

It’s not used across the internet nowadays but it occasionally gets used on local networks for configuring old hardware.

It’s been replaced by ssh and scp.

SMTP (Simple mail transfer protocol)

SMTP is a reliable protocol used to send email.

It’s typically over a TCP connection to a mail server.

It can be extended with SMTP Authentication.

Original email had none of this, so mail relays were often used for spam.

The code to the right sends an email using Python.

import smtplib
from email.mime.text import MIMEText
# assume we have a filename in textfile
with open(textfile) as fp:
    msg = MIMEText(fp.read())

msg['Subject'] = 'The contents of %s' % textfile
msg['From'] = ‘me@somewhere.org’
msg['To'] = ‘you@something.org’

# Send the message via our own SMTP server.
s = smtplib.SMTP(‘smtp.ecs.soton.ac.uk')
s.send_message(msg)
s.quit()

HTTP (Hypertext transfer protocol)

It’s a text-based protocol.

You TCP-connect to the a server, usually on port 80.

Then, you send a request - could be something like GET /something/page.html HTTP/1.1

You receive a response message with HTTP headers and your data.

Here’s what a typical response looks like:

These are the main HTTP requests:

  • GET - retrieve data
  • HEAD - like GET but no data gets sent back
  • POST - send data to the server
  • PUT - meant to change a resource on the server
  • DELETE - removes a resource

And here are the error codes (?!):

  • 1xx - info
  • 2xx - success
  • 3xx - redirection (like 301 Moved)
  • 4xx - client error (could be you’re making some sort of bad request, like 403 Forbidden or 404 Not Found)
  • 5xx - server error (like 500 Internal Error)

QUIC (Quick UDP internet connections)

It’s a UDP protocol intended to make the web faster (than using TCP).

(really, it’s a transport protocol, but it’s in user-space so it can easily be upgraded)

Lots of traffic is now QUIC, thanks to Google pushing it. The IETF is standardising it.

Uncharacteristically for UDP, it still establishes some “state”, with sessions and encryption.

RTSP (Real time streaming protocol)

Used by YouTube. Typically UDP.
