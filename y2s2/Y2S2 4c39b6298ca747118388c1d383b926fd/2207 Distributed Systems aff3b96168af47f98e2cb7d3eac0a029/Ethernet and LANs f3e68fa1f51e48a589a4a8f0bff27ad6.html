Ethernet frame

An Ethernet frame includes:

The 48-bit source and destination MAC addresses.

Tags for optional VLAN ID and frame priority, etc.

The Max Transmission Unit (MTU) for Ethernet is usually 1500 bytes.

MAC addresses

(Medium Access Control)

These are addresses at the link layer, unique to each network device.

Note that this isn’t just unique to each device - one device could have multiple network devices in it. For example, a laptop could have both a Wi-Fi card and an Ethernet controller.

MAC addresses are currently defined to be 48 bits and are extensible to 64 bits.

They need to be unique, so:

The first 24 bits are used for vendor allocations. That is, each vendor gets their own block.

The following 24 bits are assigned by the vendors themselves, to each device.

Building Ethernet networks

Typically, each host in an Ethernetwork connects to a port on a switch.

These switches receive Ethernet frames and make decisions on whether to forward the frame and to which port.

They can remember Ethernet addresses of hosts seen on each port/interface.

Hosts will usually only see packets sent to them, because of this.

Switches can store and forward Ethernet frames - this lets them perform error checking etc. before forwarding.

Some switches support advanced functions, like Quality of Service (frame prioritisation) or VLANs.

Sending IP datagrams over Ethernet

The link layer receives datagrams from the network (IP) layer and encapsulates them as the payload of an Ethernet frame.

The IP datagram has a source and destination IP address, and we need to somehow send it to the destination IP address over the Ethernet layer.

This means that we need some way to resolve an IP address into a MAC address.

ARP

We do the above using the Address Resolution Protocol - ARP.

ARP uses a link layer broadcast message.

It effectively asks “whose IP address is this??” to ff:ff:ff:ff:ff:ff, which broadcasts it to every host in the same Ethernet LAN.

Then, it’s up to the host with the target IP to respond. The sender can use this result to send the Ethernet frame on.

The “target hardware address” is empty when it gets sent, to be filled out by the host and sent back.

ARP funnies

It’s not secure.

ARP is open to spoofing - “I’m the host you’re looking for please believe me”

Note that a host can change its MAC address.

Gratuitous ARP

You can send ARP messages when you don’t need to, which lets you quickly update your banks of information for devices which have potentially changed their MAC or IP.

ARP probe

ARPs will also tell you by nature if IP addresses clash - you might get mutliple responses when you were just expecting one.

Ethernet broadcast domains

Even in a switched network, Ethernet broadcasts will flood the LAN.

In addition, ARP can only scale so far. We also have physical limits on the cable length.

This means we’ve got to keep Ethernet LANs limited to a reasonable size. In a campus this is typically one LAN per floor or building.

VLAN

The Ethernet frame includes an optional VLAN identifier. VLAN can be used to create virtual switched networks over fixed infrastructure.

This avoids the need to physically re-cable and allows you to control broadcasts to certain areas.
