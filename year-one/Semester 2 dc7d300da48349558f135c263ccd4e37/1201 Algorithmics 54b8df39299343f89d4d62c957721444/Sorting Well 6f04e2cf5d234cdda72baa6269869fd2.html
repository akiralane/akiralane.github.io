Correctness of Sorting Algorithms

Loop Invariants

A loop invariant is a property of a loop that helps us understand why an algorithm is correct. A property is a loop invariant if we can show the following:

  • Initalisation: it holds true prior to the first iteration
  • Maintenance: if it holds true for the current iteration, it must hold true for the next iteration.
  • Termination: when the loop completes, the invariant must give us a useful property that shows us the algorithm is correct (i.e. the array is sorted)

We often prove correctness of sorting algorithms using loop invariants, though it can take a surprising amount of work to do so rigorously.

A sorting algorithm is correct when:

  1. its output is in non-decreasing order - that is, the output is sorted.
  1. the items in the output are a permutation of the items in the input.

Insertion sort example

Recall that insertion sort works by keeping a sub-array of items on the left in sorted order. This sub-array is increased by moving an item in the main array into the correct position in the sub-array, continuing until the main array is sorted.

Think of it like how you'd sort a hand of cards, moving cards to the left of your hand and putting them in order.

In order to prove the correctness of Insertion Sort, we can use the following loop invariant:

  • "At the start of each iteration of the loop, the sub-array a1a2…aj−1a_1a_2\dots a_{j-1}a1​a2​…aj−1​﻿ consists of all items originally in a1a2…aj−1a_1a_2\dots a_{j-1}a1​a2​…aj−1​﻿ but in sorted order."

Merge Sort

Merge sort's basic concept: divide the array in two halves and recursively merge sort each half. then, merge the two sorted halves to obtain the solution.

procedure MergeSort(arr, start, end)
	if start < end then
		let mid = floor((start + end) / 2)
		MergeSort(arr, start, mid)
		MergeSort(arr, mid + 1, end)
		Merge(arr, start, mid, end)             // assuming some procedure Merge
	else
		return

In order to merge two sorted arrays aaa﻿ and bbb﻿ into a single sorted array ccc﻿:

  1. compare first elements of aaa﻿ and bbb﻿
  1. choose the smaller and store it in ccc﻿
  1. in the array which contained the chosen (smaller) element, move the "current element" pointer for that array along 1
  1. repeat

We know this gives us a sorted array because we know that aaa﻿ and bbb﻿ are already sorted.

Merge Sort is stable. This means that it preserves the order of two entries with the same value.

Merge Sort is not in-place. We need an additional memory space to do the merging.

It is, however, asymptotically optimal.

Quicksort

Invented by Sir Tony Hoare in 1959, quicksort is one of the most influential algorithms in computer science.

Quicksort's basic concept: divide-and-conquer by separating the array into two parts depending on whether the elements are smaller or greater than some pivot element. Recursively quicksort both parts until the array is sorted.

We do this by keeping track of the indices of the range we want to quicksort (here it's start and end but sometimes LLL﻿ and RRR﻿). Obviously when sorting the entire array, these will start as 0 and length respectively, but the recursion makes use of the smaller parts.

Quicksort is O(nlog⁡n)O(n\log n)O(nlogn)﻿ on average. It has a rare worst case of O(n2)O(n^2)O(n2)﻿ which comes from the partitioning algorithm.

Basic algorithm

  1. Pick an element - the pivot - from the array.
  1. Re-order the array so that every element less than the pivot comes before the pivot and every element greater comes after. There are multiple ways of doing this.
  1. Recursively quicksort each side of the array (g.t. and l.t. the pivot).

Picking the pivot

The worst case scenario occurs when the pivot is the smallest or largest element in the array.

This causes an inefficient partitioning, where you're sorting an array of size n−1n-1n−1﻿ on one stack and an array of size 111﻿ on the other. This is because no elements will be greater/smaller than the pivot, so all of the array will be on one side of the pivot.

So, how do we efficiently choose the pivot? Some options:

  • Choose the pivot randomly. Makes worst case unlikely, at least.
  • Choose the first element of the array.
  • Choose the median of the first, middle and last elements of the array. This increases the likelihood that the pivot is close to the median of the array.

Selection Sort

This is just for reference in case you try to find it later.

Selection sort maintains two sub-arrays.

It sorts by repeatedly finding the minimum element in the right (unsorted) array and placing it in the left (sorted) array.
