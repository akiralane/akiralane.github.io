Time Complexity

The running time of an algorithm is dependent on the size of its input, clearly. This size is dependent on the nature of the problem.

  • array size in sorting
  • number of cities in Travelling Salesman Problem

For some algorithms, the running time also depends on the input itself (i.e. some inputs will require more elementary operations). These algorithms have separate worst-case and average-case running times.

The worst case running time is a function T:N→NT:\N\to\NT:N→N﻿ where T(n)T(n)T(n)﻿ is the maximum number of elementary operations the algorithm uses on inputs of size n

The average case running time is similar, except it's the average number of elementary operations.

Example: Minimum element in an array

	m = arr[0]
	i = 1
	while (i < len(arr)):
		m = min(m, arr[i])
		i++
