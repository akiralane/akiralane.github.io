Structured Data

CSV and TSV are two popular formats for text structured data:

  • Comma Separated Values
  • Tab Separated Values

Many Unix tools are designed to work with them: cut sort grep paste join etc.

Being Careful...

Newlines

Newlines are painful.

  • Carriage Return (CR), denoted by /r, roughly means "go back to the start of the line without moving to the next line".
  • Line Feed (LF), denotes by /n, roughly means "move to the next line without moving the cursor".

Windows uses CR-LF, while Unix-like machines just use LF. Some things still just use CR (somehow).

Special Characters

Since CSV and TSV are just text, sometimes our raw data can contain characters which mess everything up because we've assigned them special meaning.

For example, trying to store the title The Lion, the Witch and the Wardrobe in CSV format will treat " the Witch and the Wardrobe" as its own entry in the next column - unless we sanitise the input.

Too basic

This format is great for certain tasks, especially those you typically come across when you want to use bash scripting. For others however, CSV/TSV isn't enough.

It can only represent data which can be put into rows and columns; this doesn't work for many types.

For example, representing a document or outlining relational data would be incredibly messy or outright impossible in CSV/TSV.

We need something which can:

  • represent entities in data
  • represent attributes of entities
  • outline relationships of entities

Markup time!

SGML: Standard Generalised Markup Language

SGML is an ISO standard for defining Markup Languages. It is therefore a superset of all markup langauges.

It separates structure from content using tags, like you're familiar with.

  • Tags: <tag>content</tag>
  • Attributes of tags: <tag attribute="value">

Aside from this, it doesn't really say much else! It is up to the user how to tag their data. Obviously, more specific standards were developed to standardise the structure.

XML was designed to be less flexible than SGML. This makes it easier and more efficient to parse. Data can be delivered without having to carry a definition of its structure.

XML, HTML and OED are all subsets of SGML.
