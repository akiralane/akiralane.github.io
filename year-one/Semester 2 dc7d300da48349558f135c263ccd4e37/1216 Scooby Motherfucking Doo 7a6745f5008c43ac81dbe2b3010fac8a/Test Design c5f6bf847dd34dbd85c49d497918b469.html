Negative as well as positive test cases are required:

The idea: we need only to test a single value from each equivalence class; testing more would be redundant.

Weak normal ECT

We define equivalence classes on the domain of input for each variable.

In doing this, we assume independence of the input variables and also make the single fault assumption:

"failures are only rarely the results of the simultaneous occurence of two or more faults"

In the above, you can see that we need 3 test cases.

Weak robust ECT

This includes weak normal, except it adds out of range test-cases for each variable.

Strong normal ECT

We cover every combination of equivalence classes for the domain of all variables.

As opposed to weak normal ECT, we assume dependence among input variables and make the multiple fault assumption.

Strong robust ECT

Same a strong normal, except we also check for all out-of-range combinations.

Boundary value testing

Complements equivalence class partitioning: we assume that the boundaries of the input and output partitions are more error-prone.

We pick values near the extremes and on the extremes, alongside a single centre-range test case.

We can add robustness by choosing values just past the extremes.
