C: Memory Allocation

In C, it's possible to access the heap, allocate some memory and assign it to a pointer. When this memory is no longer needed, you can deallocate it to return in to the heap.

The C standard library stdlib.h supports the definition of specific functions for dynamic memory management.

malloc

malloc is a function that specifies a number of bytes to reserve and returns a pointer to the beginning of the allocated storage area in memory. For example:

int *pointer, n;
n = //some number
pointer = malloc( n * sizeof(int) );

It is very possible that malloc cannot allocate the requested memory and returns a null pointer.

It is therefore good practice to check this possibility.

float *p;
p = malloc( 50 * sizeof(float));
if (p == NULL) {
	printf("malloc failed\n");
	return -1;
}

calloc

calloc takes two arguments: the number of elements to be reserved and the size of each element in bytes. Like malloc, it then returns a pointer to the beginning of the allocated storage area.

However, while we don't know what's at the location malloc gave us, calloc also sets all bytes in the given memory area to zero.

realloc

Sometimes, you need to change the amount of memory you're using to store data.

realloc is a function that changes the size of previously allocated memory (through malloc or calloc) and returns a pointer to the new block. It has the following form:

int *p;
p = malloc( 10 * sizeof(int) ); // space for an array of 10 ints
p = realloc( p, 20 * sizeof(int) ); // p now has space for 20 ints

realloc allocates space for the new memory and copies it there, instead of just changing the size of the existing memory. This means that it can have a potential performance impact if used carelessly.

free

When you have no more use for the memory you requested, you can return it to the heap using free.

float *p
p = malloc( 50 * sizeof(float));
free(p);
