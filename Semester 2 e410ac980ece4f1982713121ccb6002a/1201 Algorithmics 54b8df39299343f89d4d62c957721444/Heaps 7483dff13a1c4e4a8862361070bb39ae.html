Heaps

A min-heap is a binary tree satisfying two constraints:

  1. It is a complete tree: every level above the lowest level is fully occupied and the nodes on the lowest level are all to the left.
  1. Each child has a value greater than or equal to its parent. This is the opposite for a max-heap, where the children are less than their parents.

Adding to the heap

Heaps are easy to maintain.

To add an element to a heap:

  1. Add the element to the next available space in the tree: left to right, top to bottom.
  1. Percolate the value up the tree to maintain the correct ordering.

Priority Queues

One of the main uses of heaps is to implement a Priority Queue. Remember that a priority queue assigns a "priority" to each of its nodes and the highest priority node (typically lowest numerical value) is always pulled out first.

removeMin - this removes the highest priority node. Since the minimum element is the root of the tree, we pop the root. Then, replace it with the last element in the heap and percolate this element down, maintaining the correct ordering.

Array implementation of heaps

A surprising thing about heaps is that they can be implemented - efficiently! - using arrays. This is because the tree is complete.

If we go through the heap in order (left to right, top to bottom, one level at a time), then each node of the heap can point to an element in an array in the same order.

To navigate a heap, we note that

The root of the tree is at array location 000﻿.

The last element in the heap is at array location size() - 1

The parent of a node at location kkk﻿ is at location ⌊(k−1)/2⌋\lfloor(k-1)/2\rfloor⌊(k−1)/2⌋﻿

The children of a node at location kkk﻿ are at locations 2k+12k+12k+1﻿ and 2k+22k+22k+2﻿

This makes traversing heaps trivial.

Time complexity analysis

The two important operations are add and removeMin since we just saw that searching the heap can be done in Θ(1)\Theta(1)Θ(1)﻿ using memory addressing.

These operations work by moving elements up or down the tree by percolating a single element down or up respectively.

Percolating up/down a single level is Θ(1)\Theta(1)Θ(1)﻿, obviously.

The height of the tree is Θ(log⁡(n))\Theta(\log(n))Θ(log(n))﻿. We'll prove this later, I think.

Hence, percolating up/down is Θ(log⁡(n))\Theta(\log(n))Θ(log(n))﻿.

The remaining operations in add and removeMin are trivial since they're Θ(1)\Theta(1)Θ(1)﻿.

Thus, add and removeMin are Θ(log⁡(n))\Theta(\log(n))Θ(log(n))﻿ in the worst case.

Except, add could also require resizing the array. The amortised cost of this is low, however.

Heap sort

A priority queue suggests one very simple way to perform a sort: we simply add elements to a heap and then take them off again. They come out sorted because the add and removeMin operations effectively sort while doing what they do.

(Note that this is not an in-place sort algorithm - it uses Θ(n)\Theta(n)Θ(n)﻿ extra memory. The standard Heap Sort sorts in place, however.)

The worst-case time complexity of this is log-linear - that is, O(nlog⁡(n))O(n\log(n))O(nlog(n))﻿.

We have to add nnn﻿ elements and then remove nnn﻿ elements.

Each add/remove is O(log⁡(n))O(\log(n))O(log(n))﻿.

Extra: Standard Heap Sort

This isn't examinable.

The Standard Heap Sort (invented by John Williams in 1964) works as follows:

  1. Start with a non-sorted array
  1. Transform this into a max-heap (largest value is the root) without using additional space.
  1. Order the resulting array by repeatedly removing the maximum from the current heap
    • Each time the maximum is removed, it swaps place with the last element in the heap before the heap decreases in size.
