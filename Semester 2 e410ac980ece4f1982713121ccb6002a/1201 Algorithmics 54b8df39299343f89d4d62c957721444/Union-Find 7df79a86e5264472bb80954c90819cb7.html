We want to connect some objects together, and find which objects are connected together. Like so:

That is, we want to model equivalence classes.

Quick-Find

Here, we have an integer array of the same size as the number of objects we have. The index relates the object to the element; the index is the object's representation while the element is the object's id.

Elements ppp﻿ and qqq﻿ are equivalent/connected iff their ids are the same.

isConnected(a, b) simply checks for this.

A simple union(a, b) method would be to set the ID of a to the ID of b.

This has find of O(1)O(1)O(1)﻿ but union of O(N)O(N)O(N)﻿. What if NNN﻿ is super large and union is called frequently? We need a faster solution.

Quick-Union

We keep the integer array of size NNN﻿. This time, components are stored as multiway trees - the whole thing is a forest of trees.

The ID of an element is its parent (except when it's the root of its own tree, in which case its ID is itself). For example, id[3] = 4 means "the parent of 3 is 4".

isConnected(a, b) traverses their respective trees, checks if they share a root and returns true if they do.

union(a, b) adds the root of the tree a is part of as a direct child of the root of the tree q is part of.

Find: O(N)O(N)O(N)﻿. Union: O(N)O(N)O(N)﻿ when tree is imbalanced.

This is worse!!!! What the fuck was the point!!!!

Here's why:

Improvement: Weighted Quick-Union

Main idea: keep the tree balanced at union.

We maintain a size[] array to keep track of the number of items in each tree. When we perform a union, we always merge the smaller tree into the larger one.

The depth of any tree formed by this method is log⁡2(N)\log_2(N)log2​(N)﻿; the time complexity of find and union become O(log⁡2(N))O(\log_2(N))O(log2​(N))﻿.

Improvement: Path Compression

Observation: when we perform find in Quick-Union, we actually look for a a path from the element to its root.

Main idea: set the items on this path to be direct children of the root every time the root's calculated.

This flattens the tree considerably.

Results of the improvements
