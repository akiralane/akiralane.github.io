Keeping Trees Balanced

Deletion

Suppose we want to delete an element from a binary search tree. Remember that this tree is simulating an ordered list; if we delete a parent, we can't just delete all of its children as well. If you delete a single element in an ordered list you obviously don't want to delete a bunch of others.

It's easy if the element is held in a leaf node.

It's also not so hard if the node holding the element has a single child, since we can just point the parent of the element to the child of the element.

If an element has two children, there is a clever trick: replace that element by its successor and then remove the successor using the above procedure.

Rotations

To avoid unbalanced trees, we'd like to modify the shape. We can do this because we can make any node the root; the shape of a tree is not strict.

void rotateLeft(Node<T> e) {     // e is the element we want to rotate around

	Node<T> r = e.right;           // call e's right node r
	e.right = r.left;              // set e's right node to r's left
	if (r.left != null)            // if r's left exists,
		r.left.parent = e;           //     set the parent of r's left to be e

	r.parent = e.parent; 
	if (e.parent == null)          // if e doesn't have a parent (it was the root),
		root = r;                    //     set the root node to r instead
	else if (e.parent.left == e)   // else if e is its parent's left node,
		e.parent.left = r;           //     set the parent's left to r
	else                           // else (when e is its parent's right node)
		e.parent.right = r;          //     set the parent's right to r

	r.left = e;                    // link r with e
	e.parent = r;                  // link e with r
}

Double Rotations

If the unbalanced subtree is on the inside of the tree, we need a double rotation. These are left-right rotations and right-left rotations.

A left-right rotation is a left rotation followed by a right rotation. Holy Shit

Note that the first rotation of a left-right rotation is done about the left child of the root of the (sub)tree and the next one is done about the root.

I assume this is vice-versa for a right-left rotation.
