In Event-B, we can model a database as a set of ordered pairs. We use ↦\mapsto↦﻿ to denote ordered pairs here.

directory={mary↦287,mary↦398,john↦829,jim↦398}\text{directory}=\{mary\mapsto287, mary\mapsto398,john\mapsto829,jim\mapsto398\}directory={mary↦287,mary↦398,john↦829,jim↦398}﻿

The domain of a relation RRR﻿, denoted by dom(R)dom(R)dom(R)﻿, is given as the set of first parts, like you'd expect. {mary,john,jim}\{mary, john, jim\}{mary,john,jim}﻿

The same logic applies to the range ran(R)ran(R)ran(R)﻿. {287,398,829}\{287,398,829\}{287,398,829}﻿

The relational image of a set of elements in the domain is the set of elements in the range that are pointed to.

  • directory[{john,jim}]={829,398}\text{directory}[\{john,jim\}]=\{829,398\}directory[{john,jim}]={829,398}﻿

Partial Functions

Partial functions are a special kind of relation: any domain element points to at most one range element.

They are called partial functions because they do not have to be well-defined over the entire domain; partial functions are effectively functions from a subset of the domain.

Domain Restriction （反対の展開領域）

振り仮名：「はんたい　の　てんかい　りょういき」

Given a set AAA﻿ and a relation RRR﻿ between SSS﻿ and TTT﻿, the domain restriction of RRR﻿ by AAA﻿ is written A◃RA \triangleleft RA◃R﻿.

This means "restrict the relation RRR﻿ so that it only contains pairs whose first parts are in AAA﻿".

Remember our directory?

directory={mary↦287,mary↦398,john↦829,jim↦398}\text{directory}=\{mary\mapsto287, mary\mapsto398,john\mapsto829,jim\mapsto398\}directory={mary↦287,mary↦398,john↦829,jim↦398}﻿

We can restrict it over a set of names:

{john,jim,jane}◃directory={john↦829,jim↦398}\{john,jim,jane\}\triangleleft\text{directory}=\{john\mapsto829,jim\mapsto398\}{john,jim,jane}◃directory={john↦829,jim↦398}﻿
